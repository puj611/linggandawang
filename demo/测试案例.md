## 案例 1：直接生成模式（最快速路径）

**目的**：验证"0 问直出"功能，展示最快拿到提示词的路径

**测试提示词**：`给列表页加分页功能`

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 打开灵感大王 | 展开态显示，输入框 + 三档模式选择器可见 |
| 2 | 在输入框粘贴：`给列表页加分页功能` | 输入框显示文字，底部字符计数更新 |
| 3 | 点击「直接生成」模式 Tab | Tab 高亮为紫色，按钮文案变为「直接生成 ->」 |
| 4 | 点击「直接生成 ->」按钮 | 直接跳到结果态，显示四段式提示词 |
| 5 | 点击「复制」按钮 | 按钮变绿显示「已复制 ✓」，剪贴板中有提示词内容 |
| 6 | 点击「导出 .md」 | 浏览器下载一个 .md 文件 |

**验证点**：
- [x] 模式切换 UI 正常高亮
- [x] 直接生成不弹出任何提问
- [x] 四段式提示词均有内容
- [x] 复制/导出功能正常

## 案例 2：快速提问模式（7 题核心路径）

**目的**：验证快速模式仅问核心题目，快速到达结果

**测试提示词**：`页面太丑了，想改好看一点`

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 点击「重新提问」回到展开态 | 展开态重置，输入框清空 |
| 2 | 在输入框粘贴：`页面太丑了，想改好看一点` | 输入框显示文字 |
| 3 | 确认「快速提问」模式已选中（默认） | Tab 高亮 |
| 4 | 点击「开始提问 ->」 | 进入提问态，进度条第 1 个点（感知）亮起 |
| 5 | 第 1 题（感知）：点击「修一个看不顺眼的地方」 | 底部出现意图标签，自动跳到下一题 |
| 6 | 第 2 题（命名）：点击「颜色太多太花哨」 | 标签增加，继续下一题 |
| 7 | 第 3 题（命名）：点击「整体风格想往哪个方向走？」→ 选「极简克制（像 Apple/Linear）」 | 标签增加，继续 |
| 8 | 继续回答后续核心题（规格/执行阶段） | 快速通过，题目总数约 7 道 |
| 9 | 进入结果态 | 显示四段式提示词 |

**验证点**：
- [x] 快速模式题目数量 ≤ 7 道
- [x] 每道题回答后标签正确提取
- [x] 进度条阶段推进正常
- [x] 最终提示词包含用户选择的标签信息

## 案例 3：详细诊断模式（完整 31 题）

**目的**：验证完整诊断流程，展示深度提问能力

**测试提示词**：`我的仪表盘看起来很乱，不够专业，想改成高级感的风格`

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 回到展开态，粘贴：`我的仪表盘看起来很乱，不够专业，想改成高级感的风格` | 输入框显示文字 |
| 2 | 切换到「详细诊断」模式 | Tab 高亮，按钮文案「开始提问 ->」 |
| 3 | 点击「开始提问 ->」 | 进入感知阶段 |
| **感知阶段（约 6 题）** | | |
| 4 | p-001「今天想让页面有什么变化？」→ 点击「修一个看不顺眼的地方」 | 标签提取，跳下一题 |
| 5 | p-002「最看不顺眼的是哪个区域？」→ 点击「整体颜色不好看」 | 标签提取 |
| 6 | p-005「用一句话形容你现在的感觉？」→ 输入：`看起来太乱了，没有高级感` | 开放题回答 |
| 7 | p-006「这次改动影响多大范围？」→ 点击「改一整个页面」 | 范围确认 |
| **命名阶段（约 7 题）** | | |
| 8 | n-001「你说乱，主要是哪种乱？」→ 点击「颜色太多太花哨」 | 维度命名 |
| 9 | n-007「整体风格想往哪个方向走？」→ 点击「高级质感（像 Stripe/Linear）」 | 风格方向 |
| **规格阶段（约 8 题）** | | |
| 10 | 继续回答规格阶段问题 | 收集具体样式细节 |
| **执行阶段（约 5 题）** | | |
| 11 | 继续回答执行阶段问题 | 收集实现偏好 |
| **验证阶段（约 5 题）** | | |
| 12 | 继续回答验证阶段问题 | 收集验收标准 |
| 13 | 结果态 | 四段式提示词内容更丰富 |

**验证点**：
- [x] 五阶段进度条逐步推进
- [x] "为什么问"提示正常显示
- [x] 自定义输入功能正常
- [x] 标签随回答不断积累

## 连续测试流程（推荐操作顺序）

如果要一次性跑完所有案例，按以下顺序操作最高效：

```
1. 打开灵感大王
2. 案例 1（直接生成）→ 输入"给列表页加分页功能" → 直接生成 → 复制
3. 案例 2（快速提问）→ 重新提问 → 输入"页面太丑了" → 快速提问 → 回答 7 题 → 生成
4. 案例 8（结果编辑）→ 在案例 2 结果上编辑 + 删除标签
5. 案例 9（历史记录）→ 回到展开态 → 搜索/收藏/查看
6. 案例 3（详细诊断）→ 输入"仪表盘很乱" → 详细诊断 → 走完 31 题
7. 案例 4（跳过保护）→ 输入"登录按钮不显眼" → 快速提问 → 连续跳过
8. 案例 5（撤销回答）→ 输入"导航栏太挤" → 快速提问 → 返回上一题
9. 案例 6（截图诊断）→ 粘贴截图 → 查看诊断结果
10. 案例 7（参照图）→ 拖入参照图 → 添加到提示词
11. 案例 10（设置）→ 打开设置 → 配置 LLM
```
