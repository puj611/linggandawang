【学习工作赛道】灵感大王 · Vibe Coding 意图传达辅助工具

# 学习工作

## 1. 创意名称 + 创意介绍

**创意名称**：灵感大王

**想解决什么问题**：
vibe coding 新人常常"有感觉但说不清"——想让 AI 改 UI 却不知道该怎么描述，结果 AI 润色出来的提示词偏离原意，agent 改半天还是不对。灵感大王要做的是把这份"感觉不对"反向挤压成精准、可执行的结构化指令。

**为什么会想到做这个**：
团队自己在用 AI 辅助写前端时，反复遇到"我觉得这里太挤""看起来不够高级"这类模糊输入被 AI 随意发挥的情况。与其让 AI 猜，不如让 AI 先问，把隐性意图一层层挤出来。

**大概是什么产品**：
一款常驻桌面的悬浮窗工具（当前为 Web Demo 可体验），通过结构化反向提问 + 截图视觉诊断，把用户模糊的灵感转化为四段式提示词（动作 / 规格 / 约束 / 验证）。

## 2. 目标用户及痛点

**面向哪些用户**：
- 刚接触 vibe coding 的前端/全栈新人
- 需要频繁与 AI 协作的独立开发者、学生、产品经理
- 对视觉有感觉但缺乏设计术语表达的非设计人员

**在什么场景下使用**：
用户看到一段 UI 或想到一个功能改动时，按下热键唤起悬浮窗，输入一句模糊描述（如"首页太土了，想高级一点"），工具通过多轮提问帮他把需求结构化。

**当前痛点**：
没有灵感大王时，用户只能直接丢给 AI 一段模糊文字，得到的结果往往是 AI 的"平均值"——既不贴合真实意图，也难以复用；反复修改消耗大量 tokens 和时间，最后还得人工把提示词重新整理。

## 3. 价值与意义

**效率提升**：
把一次性模糊输入变成多轮小步对齐，显著减少 AI 返工次数。测试场景下，原本需要 5-6 轮对话才能说清的需求，可在 1 分钟内被挤压为包含动作、规格、约束、验证的完整提示词，直接复制给任意 agent 使用。

**社会价值**：
降低 vibe coding 的表达门槛，让不擅长写提示词的人也能精准驱动 AI，减少"AI 越改越偏"的挫败感，帮助更多人从"会用 AI"进阶到"用好 AI"。
