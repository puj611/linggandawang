# 灵感大王 v0.2.0 桌面安装包打包教程

> 适用版本：灵感大王 v0.2.0（含 M9.4 稳定性层 + M9.5 快速提问优化）
> 产物：`linggandawang_0.2.0_x64-setup.exe`（NSIS 安装包）
> 预估耗时：首次 8-15 分钟（拉取 Rust 依赖 + 编译 release），增量构建 2-5 分钟

---

## 〇、为什么必须在 TRAE 外部执行

TRAE IDE 内置终端使用沙箱运行 build script，会拦截子进程对 `src-tauri/target/` 目录的文件访问（写入、删除、链接），表现为：
- `error: failed to create file ... Permission denied`
- `link.exe` 阶段卡死
- `wix`/`nsis` bundling 阶段无法生成产物

**结论：在 TRAE 终端里直接跑 `npm run tauri build` 大概率会失败。** 请始终在 **独立 PowerShell 窗口** 中执行。

## 一、前置环境自检（5 分钟）

在开始打包前，先确认工具链就位。打开 **PowerShell 7 或 Windows PowerShell 5.1**（Win+R → 输入 `powershell` → 回车），依次执行：

```powershell
# 1. Node.js（需 ≥ 18，推荐 20+）
node --version
# 期望：v18.x 或 v20.x 或 v22.x

# 2. npm（随 Node 一起安装）
npm --version
# 期望：9.x 或 10.x

# 3. Rust 工具链（Tauri 编译核心）
rustc --version
cargo --version
# 期望：rustc 1.74+（推荐 1.78+），cargo 1.74+

# 4. Microsoft C++ Build Tools
where.exe cl.exe
# 期望：返回路径，例如 C:\Program Files\Microsoft Visual Studio\2022\BuildTools\VC\Tools\MSVC\...\bin\Hostx64\x64\cl.exe

# 5. WebView2 Runtime（Windows 11 自带，Win10 需安装）
Get-ItemProperty "HKLM:\SOFTWARE\WOW6432Node\Microsoft\EdgeUpdate\Clients\{F3017226-FE2A-4295-8BDF-00C3A9A7E4C5}" -ErrorAction SilentlyContinue | Select-Object pv
# 期望：返回版本号（如 120.0.2210.91）
```

### 如果某项缺失

| 工具 | 安装方法 |
|---|---|
| Node.js | 前往 [nodejs.org](https://nodejs.org/) 下载 LTS 版 |
| Rust | 前往 [rustup.rs](https://rustup.rs/) 下载 `rustup-init.exe`，默认安装 stable 工具链 |
| MSVC Build Tools | 前往 [visualstudio.microsoft.com](https://visualstudio.microsoft.com/visual-cpp-build-tools/) 下载"Build Tools for Visual Studio"，勾选 **"使用 C++ 的桌面开发"** 工作负载 |
| WebView2 | 前往 [developer.microsoft.com](https://developer.microsoft.com/microsoft-edge/webview2/) 下载"Evergreen Standalone Installer" |

## 六、执行打包（核心步骤）

### 方案 A：标准 release 打包（推荐，产物 ~20-30 MB）

```powershell
# 单条命令，会自动执行：vite build → tsc -b → rustc release → nsis bundling
npm run tauri build
```

**执行时窗口会持续输出编译日志，请耐心等待。** 正常输出会包含：

```
   Compiling tauri v2.x.x
   Compiling linggandawang v0.2.0 (D:\AI工作平台\linggandawang\src-tauri)
   ...
    Finished `release` profile [optimized] target(s) in 4m 32s
    Bundling linggandawang_0.2.0_x64-setup.exe
    Bundling linggandawang_0.2.0_x64.msi
    Bundle done
    Output: D:\AI工作平台\linggandawang\src-tauri\target\release\bundle\
```

### 方案 B：仅打 NSIS（避开 MSI 的额外依赖）

```powershell
# 等价于方案 A，但显式指定 target
npx tauri build --bundles nsis
```

### 方案 C：开发模式快速验证（不打安装包，仅看运行是否正常）

```powershell
# 仅在你想本地快速冒烟测试时使用
npx tauri dev
# Ctrl+C 退出
```

## 十、常见错误与修复

### 错误 1：`failed to create file ... Permission denied`

**原因**：在 TRAE 内置终端里执行，被沙箱拦截。
**修复**：关闭 TRAE 终端，打开 **独立 PowerShell 窗口** 重新执行。

### 错误 2：`link.exe not found` 或 `MSVC link.exe failed`

**原因**：未安装或未配置 MSVC Build Tools。
**修复**：
1. 安装 Build Tools for Visual Studio（勾选"使用 C++ 的桌面开发"）
2. 在 PowerShell 中运行 `& "C:\Program Files\Microsoft Visual Studio\2022\BuildTools\VC\Auxiliary\Build\vcvars64.bat"` 临时加载环境变量
3. 重新执行 `npm run tauri build`

### 错误 3：`error: linking with `link.exe` failed: exit code: 1120`

**原因**：通常是 keyring 链接到旧版 Windows SDK 失败。
**修复**：
```powershell
cd src-tauri
cargo clean -p keyring
cd ..
npm run tauri build
```

### 错误 4：`Bundling failed ... NSIS`

**原因**：Tauri 默认 NSIS 是机器上第一次下载二进制。
**修复**：让命令再跑一次（已下载到 `src-tauri\target\release\bundle\nsis\` 会复用）。

### 错误 5：`Out of memory` / 卡死

**原因**：Rust release 编译消耗较大内存（4-8 GB）。
**修复**：关闭其他大型应用（Chrome、IDE）；或使用方案 A 的 release profile（默认已优化）。
