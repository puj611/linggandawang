# 灵感大王 · Vibe Coding 意图传达辅助工具

> 把「感觉不对」翻译成「精准指令」。一个桌面悬浮窗，通过结构化提问把模糊灵感挤成可执行提示词。
>
> 参赛：TRAEAI 创造力大赛

## 这是什么

针对 vibe coding 新人的核心痛点——**有想法但说不清、AI 润色反而失真、agent 改不对或没改**——「灵感大王」是一个常驻桌面的悬浮窗工具。它**不润色你的提示词**，而是通过**反向审问 + 视觉诊断 + 结构化模板**，把隐性意图一步步「挤」成 agent 能精准执行的指令。

## 核心机制（与传统润色的本质区别）

| 传统润色工具 | 灵感大王 |
|------------|---------|
| 你写一段，AI 改一段 | AI 先问你，你再答 |
| AI 用统计默认值补全你没说的 | AI 暴露你没说的，让你说 |
| 产物：AI 的平均值 | 产物：你的特异性，被结构化 |
| 一次性大动作 | 多轮小步对齐 |

## 跨工具同步约定

本项目可能在 Trae IDE / Claude Code / WorkBuddy 等多个 vibe coding 工具间切换开发。同步规则见 `docs/跨工具同步方案.md`（架构师产出）。

铁律：
1. **docs/ 是单一事实源**：PRD、架构、计划书只在 docs/ 维护，各工具配置文件只做引用。
2. **.trae/ 与 .claude/ 只放工具特定配置**，不重复文档内容。
3. **每次切换工具前**：先 `git status` 确认无未提交改动；切换后先读 `docs/跨工具同步方案.md` 对齐进度。

## 开发阶段

- **Phase 1（当前）**：端侧 Demo，MVP 功能闭环（悬浮窗 + 提问引擎 + 提示词生成）
