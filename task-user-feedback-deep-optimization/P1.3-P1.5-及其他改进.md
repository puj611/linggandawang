## 2. P1.5 悬浮窗宽度可调

### 设计要点

#### `useResize` hook 返回值
```typescript
const { resizeProps, width, dragging } = useResize();
```
- `width`：从 `windowStore.width` 订阅，初始 420，clamp 在 [380, 600]
- `resizeProps`：透传给 resize 把手 div，含 `onPointerDown` / `style.cursor: ew-resize` / `role: separator` / `aria-label`
- `dragging`：拖动中状态（用于禁用过渡动画，与 `useDrag.dragging` 用法一致）

#### 拖动机制
1. `pointerdown` 记录起始 `clientX` 和当前 `width`
2. `pointermove` 计算 `dx`，更新 `windowStore.width`（`setState` 直写，触发 React 重渲染）
3. 拖动过程中 `setInterval` 每 300ms 落盘一次（避免高频写 SQLite）
4. `pointerup` 时再落盘一次最终宽度
5. 拖动期间 `document.body.style.userSelect='none'` / `cursor='ew-resize'` 防止文本选中
6. 组件卸载时兜底清理监听

#### 持久化路径
- Tauri 环境：`sqlite:linggandawang.db` → `user_preferences` 表，key=`window_width`，value=数字字符串
- 浏览器降级：localStorage，key=`sqlite_pref_window_width`
- 加载时机：`App.tsx` 启动时 `useWindowStore.loadPosition()` 已自动包含 width 加载

#### resize 把手样式
- 位置：容器右下角 `absolute right-0 bottom-0`
- 尺寸：`w-3 h-10`（宽 12px，高 40px）
- 视觉：中间一根 `w-0.5 h-4 bg-border-light` 竖条，hover 变 `bg-brand`
- `data-no-drag`：避免触发 `useDrag` 的窗口拖动
- 与 ResultPanel 的 `overflow-hidden` 配合，把手不会溢出圆角

## 3. P1.3 参照图 / 情绪板上传

### ReferenceImagePanel 组件结构

#### Props
无 props，完全自包含。从 `useFlow().addScreenshotTags` + `useAppStore().transition` 接入主流程。

#### 状态
```typescript
interface ReferenceImage {
  id: string;
  dataUrl: string;
  note: string;
}
const [images, setImages] = useState<ReferenceImage[]>([]);
const [collapsed, setCollapsed] = useState(true);   // 默认折叠
const [dragging, setDragging] = useState(false);
const [addedFlash, setAddedFlash] = useState(false); // 添加成功后按钮变绿色 1.5s
```

#### 三种添加方式
1. **拖入**：`onDrop` / `onDragOver` / `onDragLeave`，仅取 `files[0]`
2. **粘贴**：容器 `onPaste` + `tabIndex=0`，需先点击聚焦面板（**不污染全局 paste**，避免与 ScreenshotDropZone 冲突）
3. **点击选择**：隐藏 `<input type="file" accept="image/*">`，点击空状态提示区或"+ 添加一张"按钮触发

#### 上限
- 最多 3 张，`MAX_IMAGES = 3`
- 头部显示 `（n/3）` 计数

#### "添加到提示词"行为
1. 过滤出有非空备注的图：`images.filter(img => img.note.trim().length > 0)`
2. 每张转为一个 `IntentTag`：
   ```typescript
   {
     id: uuidv4(),
     label: '参照图',
     value: img.note.trim(),       // 如 "Apple 官网极简风"
     stage: 'spec',
     source_question_id: `reference-image-${img.id}`,
     source_type: 'screenshot-diagnosis',  // 复用枚举，IntentTagChips 自动黄色高亮
     confidence: 1,
     deletable: true,
     created_at: new Date().toISOString(),
   }
   ```
3. 调用 `useFlow().addScreenshotTags(tags)`（与截图诊断共享同一入口）
4. 根据当前 `appState`：
   - `expanded`：调用 `start('参照图已加入标签，继续提问')` 进入提问流程
   - `result`：`addScreenshotTags` 内部已重生成 prompt，无需额外动作
   - `question`：标签已加入，继续答下一题
   - 其他：`transition('question')`
5. 按钮显示"已加入 ✓"1.5s 后恢复

│ 🖼 参照图 / 情绪板（0/3）        展开 ▾ │  ← 折叠头部，可点击展开/收起

│ [缩略图] [备注输入框..............] [×] │  ← 已添加图列表

│                                         │

### 4.2 Ball.tsx 右键菜单增强

#### 菜单项
| 项 | 行为 | 图标 |
|---|---|---|
| 暂停驻留 / 恢复驻留 | 切换 `windowStore.alwaysOnTop`（togglePause 语义对齐窗口置顶） | ⏸ 双竖条 |
| 设置 | 调用 `appStore.openSettings()` | ⚙ 齿轮 |
| 退出 | Tauri: `getCurrentWindow().close()`；浏览器: `window.close()` | ⏏ 退出箭头（红色） |

#### 交互细节
- 菜单定位：`fixed left={e.clientX} top={e.clientY}`，跟随鼠标位置
- z-index：10001（高于 SettingsPanel 的 10000，低于 PrivacyDialog 的 20000）
- 关闭机制：document click（延迟一帧绑定避免触发菜单的同次 click 立即关闭）/ ESC
- 单击悬浮球时若菜单正打开，先关闭菜单而非展开（避免误触）
- title 提示文案从 "右键设置" 改为 "右键菜单"

#### 设计决策
1. **"暂停驻留"语义对齐 alwaysOnTop**：项目中没有独立的"驻留"概念，alwaysOnTop=true 时其他窗口无法盖住灵感大王 = "驻留"。暂停驻留 = 取消置顶 = 让其他窗口可以覆盖。菜单文案根据当前状态切换为"暂停驻留"或"恢复驻留"。
2. **退出而非隐藏**：任务要求"退出"，Tauri 的 `close()` 会触发 `close-requested` 事件并关闭窗口（应用退出）。如未来需要"最小化到托盘"，可改为 `hide()` 并配合 tray 插件。
3. **不引入第三方菜单库**：用一个 `<div>` + `useState` 实现，约 60 行代码，符合任务"简单的浮动 div"要求。

#### 导出实现
```typescript
const onExportContext = () => {
  const ctx = ctxStore.ctx;
  const json = JSON.stringify(ctx, null, 2);
  const blob = new Blob([json], { type: 'application/json' });
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  const ts = new Date().toISOString().replace(/[:.]/g, '-').slice(0, 19);
  a.download = `linggandawang-context-${ts}.json`;
  document.body.appendChild(a);
  a.click();
  document.body.removeChild(a);
  URL.revokeObjectURL(url);
};
```
- 文件名：`linggandawang-context-2026-06-25T06-30-22.json`（ISO 时间戳，文件名安全）
- 即使 ctx 为空也会导出（仅含 schema_version + timestamps）

#### 清空按钮
- 保留原有二次确认机制：首次点击变红显示"再次点击确认"，再次点击才真正清空
- 调用 `ctxStore.clear()` 写入空 schema

## 6. 设计决策说明

1. **`useResize` 独立于 `useDrag`**：两个 hook 职责分离，避免一个 hook 承担过多。`useDrag` 处理窗口移动，`useResize` 处理宽度。两个 hook 共享同一个 `windowStore.width` 订阅，互不干扰。
2. **resize 把手放在右下角而非右边缘**：把手 `w-3 h-10`（12×40px）位于右下角，不与右边缘其他内容冲突；hover 时背景变 `surface-3/50` 提示可交互。`data-no-drag` 避免误触发窗口拖动。
3. **拖动时节流持久化**：`pointermove` 高频触发，若每次都写 SQLite 会卡顿。每 300ms 落盘一次 + pointerup 最终落盘，平衡流畅度与可靠性。
4. **ReferenceImagePanel 不侵入截图诊断**：`ScreenshotDropZone` 监听全局 paste，`ReferenceImagePanel` 用 `onPaste` + `tabIndex=0` 仅在聚焦时响应，两者互不干扰。参照图独立走 `addScreenshotTags` 通道，与截图诊断共享同一意图标签入口。
5. **参照图标签复用 `screenshot-diagnosis` 类型**：避免修改 `IntentTagSource` 枚举和下游类型，`IntentTagChips` 会自动以黄色高亮显示，视觉上与截图诊断标签一致（都是"来自视觉参考"的语义）。
6. **右键菜单"暂停驻留"对齐 alwaysOnTop**：项目中无独立驻留概念，alwaysOnTop 是最接近的现有状态。文案根据当前状态动态切换，避免用户困惑。
7. **退出用 `close()` 而非 `hide()`**：任务明确要求"退出"。若未来需要最小化到托盘，可改为 `hide()` 并增加 tray 配置。
8. **context.json 预览截断 800 字符**：完整 context 可能很长（含 last_prompt 全文），预览截断 800 字符避免面板过高。"导出 .json"提供完整查看入口。
9. **导出文件名带时间戳**：`linggandawang-context-2026-06-25T06-30-22.json`，ISO 格式文件名安全，便于多次导出对比。
10. **ResultPanel 也接入动态宽度**：原 ResultPanel 固定 500px，现统一为动态宽度（默认 420，最大 600）。考虑到结果态内容较多，用户可拖宽到 600px 获得更多阅读空间。
