## 2. 数据库迁移说明

### 迁移文件命名说明
任务原文建议命名为 `V3__add_favorite_to_prompt_history.sql`，但项目既有迁移采用 `0001_/0002_/0003_` 命名规则（参见 `src-tauri/migrations/` 目录）。为保持一致性，本次新增文件命名为：

```
src-tauri/migrations/0004_add_favorite_to_prompt_history.sql
```

### 迁移内容
```sql
ALTER TABLE prompt_history ADD COLUMN favorite INTEGER NOT NULL DEFAULT 0;
CREATE INDEX IF NOT EXISTS idx_prompt_history_favorite ON prompt_history(favorite);
```

- `favorite`：0 = 未收藏（默认），1 = 已收藏
- 新增索引以加速收藏筛选查询
- 既有数据迁移后 favorite 默认为 0，不影响现有记录

## 3. sqlite.ts 改造详情

### 接口变更
```typescript
export interface PromptHistoryRecord {
  id?: number;
  title: string;
  markdown: string;
  tags: string[];
  seed_input: string;
  created_at: string;
  updated_at: string;
  favorite?: boolean;  // 新增：SQL 中为 INTEGER(0/1)，接口层统一为 boolean
}
```

### 修改的方法

| 方法 | 变更 | 说明 |
|---|---|---|
| `listPromptHistory(limit, onlyFavorite)` | 新增第二参数 `onlyFavorite = false` | SQL 层用 `WHERE favorite = 1` 过滤；localStorage 降级路径用 `item.favorite === true` 过滤；返回结果统一转换 favorite 为 boolean |
| `getPromptHistoryById(id)` | SQL 增加 favorite 字段查询 | 返回结果增加 `favorite: Number(rows[0].favorite) === 1` 转换 |

### 新增的方法

| 方法 | 签名 | 说明 |
|---|---|---|
| `searchPromptHistory(keyword, limit)` | `(keyword: string, limit = 20) => Promise<PromptHistoryRecord[]>` | 按 title / markdown LIKE 搜索（大小写不敏感）；空关键词等价于 listPromptHistory；localStorage 降级用 `includes()` 子串匹配 |
| `listFavoritePrompts(limit)` | `(limit = 20) => Promise<PromptHistoryRecord[]>` | 快捷方式，内部调用 `listPromptHistory(limit, true)` |
| `togglePromptFavorite(id, favorite)` | `(id: number, favorite: boolean) => Promise<void>` | UPDATE favorite 字段并刷新 updated_at；localStorage 降级路径原地更新 |

### deletePromptHistory 说明
任务清单中提到"新增 deletePromptHistory(id)"，但该方法在改动前已存在（`src/lib/sqlite.ts` 原 line 132-140），实现正确，无需重复添加。本次保持原实现不动。

### localStorage 降级路径
所有新增/修改方法均提供 localStorage 降级实现，保证浏览器环境（非 Tauri）下功能完整可用：
- `searchPromptHistory`：内存中 `toLowerCase().includes()` 匹配
- `togglePromptFavorite`：原地更新记录的 favorite 字段
- `listPromptHistory(onlyFavorite)`：内存过滤 `item.favorite === true`

### 新增功能

#### 4.1 搜索框
- 顶部输入框，placeholder "搜索历史提示词…"
- 输入即触发 `reload`（依赖 keyword 变化）
- 有搜索词时调用 `searchPromptHistory`，无搜索词时调用 `listPromptHistory`
- 搜索与收藏筛选可叠加：搜索结果在客户端再按 onlyFavorite 过滤

#### 4.2 收藏筛选
- 右上角"☆ 收藏"按钮，点击切换 onlyFavorite
- 激活时显示"★ 收藏中"（amber-400 高亮）
- 标题动态显示"历史记录 · 收藏"

#### 4.3 展开收起
- 默认显示 5 条
- 超过 5 条时底部显示"查看全部 (N)"按钮
- 展开后按钮变为"收起"

#### 4.4 列表项预览优化
每条记录显示两行：
- 第一行：预览文本（markdown 前 60 字符，单行截断）
- 第二行（text-tertiary，9px）：种子输入 / 标签数 / 相对时间

相对时间格式化函数 `formatRelativeTime`：
- < 1 分钟 → "刚刚"
- < 1 小时 → "X分钟前"
- < 1 天 → "X小时前"
- < 30 天 → "X天前"
- ≥ 30 天 → "X个月前"

#### 4.5 收藏切换按钮
- 每条记录右侧显示 ★/☆ 图标
- 已收藏显示 ★（amber-400），未收藏显示 ☆（text-secondary）
- 点击调用 `togglePromptFavorite(id, !favorite)` 后 reload

#### 4.6 删除按钮（hover 显示）
- 每条记录右侧最末显示 × 图标
- 默认 `opacity-0`，hover 整行时 `group-hover:opacity-100` 显示
- 点击调用 `deletePromptHistory(id)` 后 reload
- hover 时颜色变为 red-400

### 样式规范
- 沿用现有深色主题 token：`bg-surface-2`、`border-border`、`text-text-secondary`、`text-text-primary`、`text-text-tertiary`
- 收藏图标用 tailwind 默认色板 `text-amber-400`（配置使用 extend，默认色板可用）
- 删除 hover 用 `hover:text-red-400`
- 未引入任何新依赖（图标用 Unicode ★☆× 字符）

## 6. 注意事项

1. **迁移文件命名**：采用 `0004_` 前缀保持与 `0001_/0002_/0003_` 一致，未采用任务原文的 `V3__` 命名
2. **favorite 字段类型转换**：SQL 层为 INTEGER(0/1)，接口层统一为 boolean，所有查询方法在 map 时用 `Number(r.favorite) === 1` 转换
3. **搜索 + 收藏叠加**：搜索结果在客户端再按 onlyFavorite 过滤，因为 `searchPromptHistory` 不接受 onlyFavorite 参数（保持方法职责单一）
4. **竞态说明**：搜索框输入会触发多次 reload，SQLite 查询很快，未做 debounce；若后续发现输入卡顿可加 debounce
5. **localStorage 降级**：所有新方法在浏览器环境（非 Tauri）下均有降级实现，功能完整
6. **删除按钮无二次确认**：当前删除直接执行，未加确认弹窗（符合"hover 显示"的轻量交互定位）；如需防误删可后续在 P1.3 增加 confirm
7. **既有 deletePromptHistory**：任务清单提到"新增"，但该方法改动前已存在且实现正确，本次保持不动，未重复添加
