## 3. QuestionLoader 改造详情

### 新增方法
| 方法 | 签名 | 说明 |
|---|---|---|
| `getScene()` | `() => string` | 返回当前激活场景（默认 'frontend-ui'） |
| `setScene(scene)` | `(scene: string) => void` | 切换激活场景 |

### 内部改造
- `private bank: QuestionBank | null` → `private banks: Map<string, QuestionBank>`
- 新增 `private activeScene: string = 'frontend-ui'`（默认前端，向后兼容）
- `loadAllBanks()` 方法：循环加载所有内置场景库
- `getQuestionById()` 增强：先查当前场景，找不到则跨场景查找（支持 seedRouter 跨场景路由）

### 向后兼容性
- `load()` 默认返回 frontend-ui 场景的问题库 → 原有测试不变
- `getBank()` / `getQuestionsByStage()` 行为不变
- `reload()` / `setUserBank()` / `clearUserBank()` 行为不变（清除所有缓存）
- 用户覆盖（userBankText）仅对 frontend-ui 场景生效

## 4. seedRouter 改造详情

### 新增函数
```typescript
export type SceneType = 'frontend-ui' | 'backend-api';
export function detectScene(seed: string): SceneType;
```

### 场景检测逻辑
- 后端关键词（40+）：接口/api/后端/数据库/sql/查询/缓存/redis/mysql/jwt/认证...
- 前端关键词（24+）：页面/按钮/颜色/布局/样式/ui/间距/圆角/字体...
- 后端命中数 > 前端命中数 → `backend-api`
- 否则 → `frontend-ui`（默认）

### routeSeedWithClusters 返回值
新增 `scene: SceneType` 字段，原有字段保持不变。

### 后端路由（9 条）
| 关键词 | 目标题目 | 说明 |
|---|---|---|
| restful/rpc/graphql/grpc | b-005 | 接口路径风格 |
| 存储选型/redis/mongo | b-006 | 数据存储选型 |
| 状态机/工作流/分布式事务 | b-007 | 业务逻辑复杂度 |
| 数据模型/表结构/建表 | b-009 | 数据模型关系 |
| jwt/认证/oauth/session | b-013 | 认证方式 |
| 分页/cursor/offset | b-012 | 分页排序约定 |
| 错误码/错误处理/状态码 | b-011 | 错误码体系 |

## 6. 注意事项

1. **bank.yaml 未修改**：前端问题库完全保持原样
2. **schema.json 未修改**：scene 枚举已包含 backend-api，无需变更
3. **ID 前缀设计**：后端题目使用 `b-` 前缀（如 b-001），与前端 p/n/s/e/v 前缀不冲突，合并时无 ID 碰撞

5. **用户覆盖**：仅对 frontend-ui 场景生效，后端场景不支持用户覆盖（后续可扩展）
