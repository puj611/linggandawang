## 4. 架构亮点

### 4.1 撤销机制设计
- 历史栈采用快照模式，每次 answer/skip 前调用 `pushHistory()` 保存完整状态
- 栈大小限制 20，超出时丢弃最旧记录，避免内存膨胀
- `undoLastAnswer` 恢复 current/stage/askedIds/answers/intentTags/consecutiveSkips 全部状态
- UI 层通过 `canUndo` 标记动态显示「↻ 上一题」按钮

### 4.2 分块编辑设计
- 每段独立维护 editing/draft 状态
- useEffect 监听外部 content 变化（如删除标签重生成时）自动同步 draft
- 编辑态切换 textarea，失焦自动保存，提供「✎ 编辑」「✓ 保存」「✕ 取消」三按钮
- PromptGenerator.updateSection 返回新 PromptResult 副本，不可变更新

### 4.3 多场景问题库
- bank-backend.yaml 与 bank.yaml 并行存在
- QuestionLoader 用 Map<scene, QuestionBank> 管理多场景
- seedRouter.detectScene 根据关键词自动路由（40+ 后端关键词、24+ 前端关键词）
- 向后兼容：默认 frontend-ui 场景
