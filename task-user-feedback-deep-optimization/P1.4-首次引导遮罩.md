## 1. 问题背景

用户反馈：打开悬浮窗看到空白输入框，完全不知道第一句话怎么写。
解决思路：首次启动时展示 3 步引导遮罩，告知核心交互（唤起 → 输入 → 提问 → 生成）与热键。
非首次启动时不打扰；提供"重新查看新手引导"入口在设置面板。

## 4. App.tsx 改造详情

### 新增状态
```typescript
const [onboardingVisible, setOnboardingVisible] = useState<boolean | null>(null);
```
`null` = 未决定（启动时遮罩不渲染，避免闪烁）；`true` = 显示；`false` = 隐藏。

### 启动检查
在原有 `useEffect` 加载流程末尾追加：
```typescript
const seen = await getPreference<string>(ONBOARDING_SEEN_KEY, '');
setOnboardingVisible(seen !== '1');
```
读取失败时降级为不显示（`setOnboardingVisible(false)`），避免阻塞用户使用。

### 完成回调
```typescript
const onOnboardingComplete = async () => {
  setOnboardingVisible(false);
  await setPreference(ONBOARDING_SEEN_KEY, '1');
};
```
点击「开始使用」和「跳过引导」均触发此回调，统一标记为已看过。

### 跨组件事件监听
```typescript
useEffect(() => {
  const handler = () => setOnboardingVisible(true);
  window.addEventListener(ONBOARDING_SHOW_EVENT, handler);
  return () => window.removeEventListener(ONBOARDING_SHOW_EVENT, handler);
}, []);
```
采用 `window CustomEvent` 而非修改 appStore，避免引入新的全局状态耦合。

### 渲染
```tsx
{onboardingVisible && <OnboardingOverlay onComplete={onOnboardingComplete} />}
```

## 5. SettingsPanel.tsx 改造详情

### 新增处理函数
```typescript
const onShowOnboarding = () => {
  void setPreference(ONBOARDING_SEEN_KEY, '');     // 重置标记
  window.dispatchEvent(new CustomEvent(ONBOARDING_SHOW_EVENT));  // 派发事件
  closeSettings();                                  // 关闭设置面板
};
```

### 新增按钮位置
位于「刷新问题库」与「清空上下文」之间，属于帮助类操作。
按钮文案：「重新查看新手引导」，配问号图标。
辅助说明：「重看 3 步使用流程和热键」。

### 与 App.tsx 的约定
- 事件名常量 `ONBOARDING_SHOW_EVENT = 'show-onboarding'`
- 偏好键常量 `ONBOARDING_SEEN_KEY = 'onboarding_seen'`
- 两个文件各自定义同名常量（避免引入共享常量文件造成过度抽象）

## 7. 设计决策说明

1. **z-index 层级 15000**：低于 PrivacyDialog（20000）确保隐私确认优先；高于 SettingsPanel（10000）和 ExpandedCard（9999）确保引导遮罩盖过其他弹层
2. **不修改 appStore**：用 `window CustomEvent` 跨组件通信，避免为单一一次性弹窗污染全局 store
3. **状态三态 `null | true | false`**：`null` 表示「启动时尚未读取偏好」，此时 `{onboardingVisible && ...}` 不渲染，避免遮罩闪现后消失
4. **「跳过引导」与「开始使用」统一标记为已看过**：避免用户跳过后每次启动都弹出，体验更克制；想重看可走设置面板入口
5. **「重新查看」时重置标记为空字符串**：即使用户中途关闭引导遮罩，下次启动仍会再次显示，符合"重新引导"语义
6. **热键文案硬编码**：组件是静态展示场景，避免从 useHotkey 动态读取造成循环依赖；如未来支持热键自定义，可在引导页读取实际热键后调整
