# 灵感大王服务维度分析与模糊形容词映射

> 文档用途：为本地规则引擎升级提供方向决策与词表基础  
> 当前版本：v1.0  
> 基于题库：`src/question-bank/bank.yaml` (frontend-ui-v1)

---

## 一、当前服务维度诊断

### 1.1 现状

灵感大王当前题库 `scene: frontend-ui`，五阶段流程：

- **perceive 感知阶段**：识别用户想干嘛（修 UI / 加细节 / 复制特性）
- **name 命名阶段**：把模糊感受翻译成具体设计维度（乱/丑/高级/看不清/呆板/不统一）
- **spec 规格阶段**：把维度量化为数值（间距、圆角、字号、配色、对比度、阴影、宽度）
- **execute 执行阶段**：确定实现方式（Tailwind / 组件 props / 全局 CSS / Design Token）
- **verify 验证阶段**：定义验收标准（截图对比 / 清单走查 / WCAG 对比度）

这是一个**非常完整的"前端 UI 视觉诊断与改进"工作流**。

### 1.2 "场景太固定"的真正原因

不是领域选错了，而是**输入解析层太薄**：

1. `seedRouter.ts` 只有 12 组关键词，且一对一映射
2. 用户输入"我想做个厉害的" → 没有命中任何关键词 → 走默认流程 → 拿到固定选项
3. 用户说"好看""舒服""大气"等模糊形容词时，系统无法展开成多个候选维度

**结论**：不需要换领域，需要把**模糊输入 → 多维意图标签**的映射做厚。

## 二、可选服务维度对比分析

| 方案 | 定位 | 优势 | 劣势 | 与现有资产匹配度 |
|------|------|------|------|----------------|
| **A. 继续做前端 UI 视觉诊断（窄而深）** | 帮前端/设计师快速定位 UI 问题并给出可执行方案 | 题库成熟、流程闭环、差异化强 | 受众较窄 | **极高** |
| **B. 扩展为通用 AI 提示词生成器（宽而浅）** | 什么提示词都能生成 | 受众广 | 竞争惨烈、PromptGenerator 需重写、失去特色 | 低 |
| **C. 专注 Design Token / 设计系统生成** | 从需求直接输出一套 tokens | 商业价值高 | 需要大量新增题库、当前 execute 阶段偏弱 | 中 |
| **D. 转为代码实现助手（直接出 Tailwind/CSS）** | 问完直接生成代码片段 | 闭环最强 | 技术栈绑定、需接入代码生成模块 | 中 |

### 推荐结论：**方案 A，但做两件事升级**

1. **横向扩展可识别的模糊形容词**：让用户能"像人一样说话"
2. **纵向深化每个维度的追问策略**：根据用户描述动态生成澄清问题，而不是固定选项

原因：

- 现有 30 道题、5 阶段流程是宝贵资产，推倒重来不划算
- 前端 UI 诊断是真实痛点，市场上缺乏"小而美"的轻量工具
- "厉害""高级""舒服"这些词在 UI 领域完全可以映射到具体维度
- 后续 LLM 增强时，也有清晰的 fallback 边界

## 四、词表接入规则引擎的策略

### 4.1 替换 `seedRouter.ts` 的匹配逻辑

从：
```ts
if (text.includes('高级')) return n-003
```

升级为：
```ts
// 1. 提取所有命中的形容词簇
const clusters = matchAdjectiveClusters(text)

// 2. 聚合维度标签权重
const tagScores = aggregateTagScores(clusters)
// 例如：{ "痛点: 不够高级": 1.4, "维度: 留白不足": 0.9, "维度: 对比度不足": 0.7 }

// 3. 返回最高分的若干标签，供 Selector 使用
return { tags: topTags(tagScores, 3), matchedKeywords }
```

### 4.2 改造 `Selector.ts` 的选题逻辑

当前只比较 `trigger_tags` 是否包含。升级后：

1. 每个问题不仅有 `trigger_tags`，还可以有 `trigger_clusters`（形容词簇）
2. 根据 `tagScores` 给每道题打分
3. 返回得分最高的未问问题
4. 如果没有明确高分项，进入**澄清追问**而不是默认选项

### 4.3 新增动态澄清问题生成

当模糊输入命中多个高权重维度时，不直接跳到 spec，而是生成一道 name 阶段的动态题：

```ts
function buildClarificationQuestion(topTags: TagScore[]): Question {
  return {
    id: 'dynamic-clarify',
    stage: 'name',
    text: `你说的"${originalInput}"，更偏向哪一种感觉？`,
    type: 'single-choice',
    options: topTags.map(t => ({
      label: tagToLabel(t.tag),
      value: t.tag,
      tags: [t.tag]
    }))
  }
}
```

### 4.4 `intent_extraction` 增强

每道题的 `intent_extraction.keywords` 应该从单个关键词扩展为**簇 + 权重**：

```yaml
intent_extraction:
  clusters:
    - cluster_id: "高级感"
      weight: 1.0
      extract_tags: ["痛点: 不够高级"]
    - cluster_id: "科技感"
      weight: 0.8
      extract_tags: ["维度: 配色", "维度: 缺动效"]
```
