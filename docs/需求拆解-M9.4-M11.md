## R1: M9.4 LLM 稳定性层

**目标**：LLM API 不稳定时自动降级，保证用户流程不被中断。

### R1-S1: 会话级熔断器

| 项 | 内容 |
|---|---|
| **文件** | `src/lib/llm/circuit-breaker.ts`（新建） |
| **逻辑** | 连续 3 次失败 → 熔断（暂停 LLM 调用 60s）→ 60s 后半开（允许 1 次试探）→ 成功则恢复，失败则继续熔断 |
| **状态** | `closed` / `open` / `half-open` |
| **存储** | 内存（会话级，不持久化） |
| **验收** | ① 模拟连续 3 次失败后第 4 次调用直接抛 `CircuitBreakerOpenError` ② 等待 60s 后允许 1 次试探 ③ 试探成功后恢复正常调用 |
| **测试** | `src/lib/llm/__tests__/circuit-breaker.test.ts`：3 个用例（熔断触发/半开恢复/半开失败） |

### R1-S2: 错误分类

| 项 | 内容 |
|---|---|
| **文件** | `src/lib/llm/error-types.ts`（新建） |
| **逻辑** | 7 类错误：`network`（fetch 失败）/ `auth`（401/403）/ `rate_limit`（429）/ `server`（5xx）/ `timeout`（AbortError）/ `format`（JSON 解析失败）/ `unknown` |
| **验收** | ① 每种 HTTP 状态码映射到正确分类 ② 网络错误归 `network` ③ 超时归 `timeout` ④ 未知错误归 `unknown` |
| **测试** | `error-types.test.ts`：7 个用例（每类 1 个） |

### R1-S3: 指数退避重试

| 项 | 内容 |
|---|---|
| **文件** | `src/lib/llm/retry.ts`（新建） |
| **逻辑** | `withRetry(fn, { maxRetries: 2, baseDelay: 1000 })`；仅重试 `server`/`network`/`timeout`；`auth`/`rate_limit`/`format` 不重试；退避：1s → 2s → 4s |
| **验收** | ① 5xx 错误重试 2 次后成功 ② 401 错误不重试直接抛出 ③ 429 不重试（避免加剧限流） ④ 重试间隔符合指数退避 |
| **测试** | `retry.test.ts`：4 个用例 |

### R1-S4: API Key 脱敏日志

| 项 | 内容 |
|---|---|
| **文件** | `src/lib/llm/log.ts`（新建） |
| **逻辑** | `maskApiKey('sk-abcdef123456')` → `sk-***123456`；`logRequest(method, url, maskedKey)` |
| **验收** | ① Key 长度 <8 时全掩码 ② Key 长度 ≥8 时显示前 3 + 后 6 ③ 日志不含完整 Key |
| **测试** | `log.test.ts`：3 个用例 |

### R1-S5: 熔断器集成到适配器

| 项 | 内容 |
|---|---|
| **文件** | `src/lib/llm/openai-adapter.ts`（修改） |
| **逻辑** | `chat()` 和 `chatStream()` 调用前检查熔断器状态；调用后根据结果更新熔断器；用 `withRetry` 包裹实际 fetch |
| **验收** | ① 适配器正常工作时熔断器不影响调用 ② 连续失败后熔断器阻断调用 ③ 恢复后自动闭环 |
| **测试** | 集成测试 2 个用例 |

### R1-S6: 降级提示 UI

| 项 | 内容 |
|---|---|
| **文件** | `src/components/LLMConfigSection.tsx`（修改） |
| **逻辑** | 熔断器 open 时显示"LLM 暂时不可用，已切换为纯规则引擎模式"提示条 |
| **验收** | ① 熔断时显示提示 ② 恢复后提示消失 ③ 提示不影响规则引擎正常工作 |
| **测试** | 手动验证（UI 交互） |

| **验收** | ① 正常需求文本拆出 3-8 个子任务 ② 每个子任务有 title/description/status ③ 异常输入有兜底 |
