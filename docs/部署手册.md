# 灵感大王 · 部署手册

> 本文档覆盖三种部署方式：开发环境、前端独立构建、桌面安装包打包。

## 一、环境要求

### 必需工具

| 工具 | 最低版本 | 推荐版本 | 用途 |
|---|---|---|---|
| Node.js | 18.x | 20.x LTS | 前端构建 |
| npm | 9.x | 10.x | 包管理 |
| Rust | 1.77.2 | 1.78+ stable | Tauri 后端编译 |
| MSVC Build Tools | VS 2022 | VS 2022 | Rust 链接器（link.exe） |

### 可选工具

| 工具 | 用途 |
|---|---|
| PowerShell 7 | 执行打包命令（Win10 自带 5.1 也可） |
| Git | 版本控制 |

### Windows 特殊要求

- **WebView2 Runtime**：Windows 10 1803+ / Windows 11 自带；低版本需手动安装
- **MSVC Build Tools**：安装时勾选"使用 C++ 的桌面开发"工作负载

### 环境自检

```powershell
node --version     # 期望 v18+ 或 v20+
npm --version      # 期望 9+ 或 10+
rustc --version    # 期望 1.77.2+
cargo --version    # 期望 1.77.2+
where.exe cl.exe   # 期望返回路径
```

## 二、方式一：开发环境

适合本地开发和调试，支持前端 HMR 热更新。

### 2.1 仅前端开发（浏览器模式）

```powershell
cd "d:\AI工作平台\linggandawang"
npm install
npm run dev
# 浏览器打开 http://localhost:5173
```

**限制**：浏览器模式下 Tauri 命令不可用，SQLite/文件 I/O/API Key 会降级为 localStorage 或内存模式。

### 2.2 Tauri 桌面开发模式（完整功能）

```powershell
cd "d:\AI工作平台\linggandawang"
npm install
npm run tauri dev
```

**行为**：
- Vite 启动开发服务器
- Tauri 编译 Rust 后端（首次 2-5 分钟，增量秒级）
- 自动打开桌面窗口
- 前端代码变更触发 HMR
- Rust 代码变更触发重新编译

**注意**：
- 全局热键 `Alt+Shift+Space` 切换窗口显示/隐藏
- `vite.config.ts` 已配置忽略 `src-tauri/` 目录，避免 Rust 文件变更触发前端重载

## 三、方式二：前端独立构建

适合生成 Web Demo 或部署到静态服务器。

```powershell
cd "d:\AI工作平台\linggandawang"
npm install
npm run build
# 产物：dist/ 目录
```

**产物结构**：
```
dist/
├── index.html
├── assets/
│   ├── *.js    # 约 328KB
│   └── *.css   # 约 15KB
└── favicon.svg
```

**部署方式**：
- 本地预览：`npm run preview`（http://localhost:4173）
- 静态服务器：将 `dist/` 内容上传到任意 HTTP 服务器
- 自包含 HTML：使用 `scripts/build-deliverables.cjs` 生成单文件 Demo

**限制**：同方式一，无 Tauri 后端能力。

## 四、方式三：桌面安装包打包

适合分发 Windows 安装包。详细教程见 [deliverables/打包教程/打包教程.md](../deliverables/打包教程/打包教程.md)。

### 4.1 快速打包

```powershell
# 必须在独立 PowerShell 窗口执行（非 TRAE 内置终端）
cd "d:\AI工作平台\linggandawang"
npm install
npm run tauri build
```

### 4.2 产物路径

```
src-tauri/target/release/bundle/
├── nsis/
│   └── linggandawang_0.2.0_x64-setup.exe   # NSIS 安装包（约 20MB）
└── msi/                                     # 可选，方案 A 会额外生成
    └── linggandawang_0.2.0_x64.msi
```

### 4.3 仅打 NSIS（跳过 MSI）

```powershell
npx tauri build --bundles nsis
```

### 4.4 关键注意事项

1. **不能在 TRAE 内置终端执行**：沙箱会拦截 build script 子进程文件访问
2. **首次编译耗时**：8-15 分钟（Rust 依赖编译）
3. **增量编译**：2-5 分钟（仅编译变更的 crate）
4. **内存需求**：Rust release 编译需要 4-8 GB 可用内存
5. **产物大小**：NSIS 安装包约 20MB，包含 WebView2 引导程序

## 五、常见问题

### Q1: `npm run tauri dev` 报 `link.exe not found`

**原因**：未安装 MSVC Build Tools 或未配置环境变量。

**解决**：
1. 安装 Build Tools for Visual Studio 2022
2. 勾选"使用 C++ 的桌面开发"工作负载
3. 重启终端

### Q2: `npm run tauri build` 报 `Permission denied`

**原因**：在 TRAE 内置终端中执行，被沙箱拦截。

**解决**：打开独立 PowerShell 窗口（Win+R → `powershell` → 回车）重新执行。

### Q3: 浏览器模式下 API Key 配置无法保存

**原因**：浏览器模式无法调用 keyring（Windows Credential Manager）。

**解决**：API Key 存储需要 Tauri 桌面环境，浏览器模式下配置不会持久化。

### Q4: `npm run tauri dev` 窗口不显示

**原因**：可能被全局热键隐藏。

**解决**：按 `Alt+Shift+Space` 切换窗口显示。

### Q5: 打包后安装的应用白屏

**原因**：WebView2 Runtime 未安装。

**解决**：
- Windows 10 1803+ / Windows 11 自带 WebView2
- 低版本 Windows 需从 [developer.microsoft.com](https://developer.microsoft.com/microsoft-edge/webview2/) 下载安装

### Q6: `cargo build` 报 `keyring` 相关错误

**原因**：keyring v3 API 变更。

**解决**：确认 `Cargo.toml` 中 `keyring = "3"`，代码中使用 `delete_credential()` 而非 `delete_password()`。

## 六、Trae Work 远程开发注意事项

使用 Trae Work 远程控制电脑时：

| 操作 | 可行性 | 说明 |
|---|---|---|
| 代码编辑 | ✅ | 完全可行 |
| `npm run dev` | ✅ | 前端开发服务器可远程启动 |
| `npm run test` | ✅ | 单元测试可远程运行 |
| `npm run build` | ✅ | 前端构建可远程执行 |
| `npm run tauri dev` | ⚠️ | 可启动，但无法看到桌面窗口（需远程桌面） |
| `npm run tauri build` | ⚠️ | 可执行，但 TRAE 终端沙箱可能拦截（建议用独立 PowerShell） |
| GUI 交互测试 | ❌ | 无法看到桌面窗口、点击交互 |
| 安装包冒烟测试 | ❌ | 需要桌面环境 |
