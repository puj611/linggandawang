## 三、测试视角结论（严过关 · QA 工程师）

### 问题清单

| 严重度 | 问题 | 位置 |
|--------|------|------|
| 🔴 高 | **测试覆盖严重不足**。整个 `src/` 仅有 `src/test/setup.ts` 一个测试配置文件，`__tests__/` 目录存在于 engine/、lib/、stores/ 下但无实际测试文件 | `src/test/setup.ts` |
| 🔴 高 | `QuestionEngine` 是最核心的状态机（638行），包含 tagScores 聚合、历史栈 undo、LLM 置信度计算等复杂逻辑。无测试意味着任何回归都不可感知 | `src/engine/QuestionEngine.ts` |
| 🟡 中 | `seedRouter.ts` 的否定词识别逻辑（`detectNegation`）有 63 行，处理中英文混合语序（"不要圆角"/"圆角不要"）。这类 NLP 边界场景极需测试用例 | `src/engine/seedRouter.ts:29-63` |
| 🟡 中 | `normalizeAnalysis` 和 `normalizeOutput` 是 LLM 输出的容错解析，已导出但未编写测试。LLM 输出格式多变，是 bug 高发区 | `src/engine/LLMIntentAnalyzer.ts:171`, `src/engine/SmartFollowup.ts:427` |
| 🟡 中 | `withRetry` 指数退避逻辑无测试。重试次数、退避间隔、不可重试错误的立即抛出均未验证 | `src/lib/llm/withRetry.ts` |
| 🟢 低 | Rust 侧 `image.rs` 的 JPEG/WebP 尺寸解析是手写的二进制解析器，无测试覆盖。畸形文件可能导致异常行为 | `src-tauri/src/commands/image.rs:188-271` |
| 🟢 低 | `vectorStore.ts` 的余弦相似度计算和嵌入管道无测试 | `src/lib/vectorStore.ts` |

### 风险点

1. **核心引擎零测试** = 回归风险极高，任何改动都可能静默破坏提问流程
2. **LLM 适配器的错误分类/熔断器逻辑无测试**，生产环境故障时行为不确定
3. **Rust 后端图片解析器无 fuzzer**，畸形输入可能触发异常
