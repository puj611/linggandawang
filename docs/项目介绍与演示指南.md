## 一、项目介绍

### 1.1 一句话定位

**把「感觉不对」翻译成「精准指令」的桌面悬浮窗工具。**

灵感大王通过反向审问 + 视觉诊断 + 结构化模板，把 vibe coding 新人脑子里模糊的灵感，一步步「挤」成 agent 能精准执行的指令。

### 1.2 解决什么痛点

| 痛点 | 场景 | 灵感大王怎么解 |
|---|---|---|
| **有想法但说不清** | "我想做个登录页，但要好看" → AI 生成的完全不是想要的 | 31 题反向提问，逼你说出"好看"到底是什么 |
| **AI 润色反而失真** | 写一段话让 AI 润色，结果被改成了"AI 平均值" | 不润色你的话，而是问你话，产物是你的特异性 |
| **agent 改不对** | 给 agent 说"优化一下 UI"，改了但没改到点上 | 生成包含量化规格的指令（间距 16px、WCAG AA 4.5:1） |
| **模糊评价无法执行** | "太挤了/不够高级/有点乱" → agent 无从下手 | 评价词典把模糊形容词翻译为可量化规格 |

### 1.3 核心机制

```
用户输入种子需求
    ↓
种子路由（关键词 + 形容词簇匹配）
    ↓
三档提问模式选择
  ├─ 直接生成（0 题）：立即生成提示词
  ├─ 快速提问（7 题）：核心规格确认
  └─ 详细诊断（31 题）：深度意图挖掘
    ↓
反向提问状态机（5 阶段）
  perceive → name → spec → execute → verify
    ↓
评价词典展开（模糊→量化）
    ↓
PromptGenerator 五段结构化输出
  动作 / 规格 / 约束 / 验证
    ↓
复制 / 导出 / 历史
```

### 1.4 与同类产品的差异

| 产品 | 生成方式 | LLM 依赖 | 交互方式 | 灵感大王差异点 |
|---|---|---|---|---|
| AIPRM | 纯模板 | 无 | 选模板+填变量 | 我们是问答交互式生成 |
| PromptBox | 人工编写 | 无 | 管理+快捷键 | 我们是自动生成 |
| OpenAI Meta-Prompt | LLM 直驱 | 强依赖 | 单轮描述 | 我们有 31 题多轮收集 |
| FlowGPT | 社区贡献 | 无 | 浏览+测试 | 我们是私有化本地工具 |

**核心差异**：灵感大王的"交互式问答 + 评价词典 + 规则引擎"组合，在"可控性"和"智能性"之间取得了独特平衡。

## 二、技术亮点

### 2.1 评价词典：模糊→量化的翻译引擎

**文件**：`src/engine/评价词典.ts`

把用户模糊评价翻译为可量化规格建议，是本项目最核心的创新点。

| 模糊评价 | 自动转化的规格建议 |
|---|---|
| 太挤 | 卡片内边距 8px→16px；卡片间距 12px→20px |
| 不够高级 | 留白≥16px + 对比度≥4.5:1 + 过渡动效 200ms |
| 乱 | 统一到 8px 网格；字号 3 级；颜色主色+1 强调色 |
| 看不清 | WCAG AA 4.5:1；正文最小 12px |
| 呆板 | 进出场 200ms cubic-bezier(0.4,0,0.2,1)；hover scale(1.02) |
| 不统一 | 设计 token：间距 8/12/16/24；圆角 4/8/12 |
| 刺眼 | 主色 HSL S 降 15%；对比度收敛 4.5-7:1 |

每条规则带 `followupHint`（追问提示），如"丑具体是指颜色、对齐、字号还是圆角？"

### 2.2 三档提问模式

| 模式 | 题数 | 耗时 | 适用场景 |
|---|---|---|---|
| 直接生成 | 0 | <1s | 目标明确，只需强化 |
| 快速提问 | 7 | 1-2min | 有方向，需确认关键规格 |
| 详细诊断 | 31 | 5-8min | 不确定方向，需深度诊断 |

### 2.3 形容词簇路由

**文件**：`src/question-bank/adjective-clusters.yaml`

支持把"太挤/有点乱/不够清爽"等修饰性形容词，通过簇匹配机制加权到 dimension_tags，并支持动态澄清题生成。

路由来源：`keyword`（关键词命中）/ `cluster`（形容词簇命中）/ `fallback`（兜底）

### 2.4 engine 层零依赖纯函数库

提问引擎是纯 TypeScript 函数库，**零外部依赖**，可在 Node 环境独立测试。不依赖 React、Tauri 或任何 npm 包。这保证了引擎的可移植性和可测试性。

### 2.5 安全基线

- API Key → keyring → Windows Credential Manager，绝不落盘
- 敏感文件黑名单（30+ 模式：`.env*`、`id_rsa`、`*.pem`、`*.key`、`.ssh`、`.aws` 等）
- SSRF 防护：拒绝 `10.*` / `172.16-31.*` / `192.168.*` / `169.254.*` 等内网地址
- 严格 CSP：`default-src 'self'; connect-src 'self' https:; object-src 'none'`
- 截图数据仅存内存，不写磁盘

### 2.6 LLM 适配器（统一 OpenAI 兼容格式）

| 服务商 | 默认模型 | 特色 |
|---|---|---|
| DeepSeek | deepseek-chat | 性价比高，中文理解强 |
| OpenAI | gpt-4o-mini | 综合能力最强 |
| 通义千问 | qwen-plus | 国内访问快 |
| 小米 MiMo | mimo-v2.5-pro | 小米 MiMo 模型，OpenAI 兼容 |
| 自定义 | 用户填写 | 兼容 OpenAI 格式 |

支持流式响应（SSE 解析）、连接测试、SSRF 防护。

### 2.7 Tauri 桌面端优势

- **安装包仅 3.3MB**（NSIS 单 exe，无额外依赖）
- **全局热键** `Alt+Shift+Space` 唤起/隐藏
- **无框透明窗口**，悬浮球常驻桌面
- **位置记忆**：窗口位置持久化到 `~/.linggandawang/window-position.json`
- 16 个 Rust 命令覆盖全部平台能力

## 三、演示脚本

### 3.1 演示前准备

```
1. 确保 DeepSeek API Key 已配置（设置页 → AI 模型配置 → 设置密钥）
2. 准备一个目标项目文件夹（用于项目感知演示）
3. 准备一张 UI 截图（用于截图诊断演示）
4. 关闭其他无关窗口，避免干扰
```

### 3.2 演示流程（建议 5 分钟）

#### 场景一：直接生成模式（30 秒）

**目的**：展示最快路径——从模糊需求到结构化提示词

```
1. Alt+Shift+Space 唤起悬浮球
2. 单击展开输入卡
3. 输入："帮我做一个用户登录页面"
4. 选择"直接生成"模式
5. 点击"开始提问 →"
6. → 0.5 秒内进入结果页，展示四段提示词
7. 点击"复制"按钮
8. 展示：提示词包含动作/规格/约束/验证四段
```

**讲解要点**：
- "注意，没有问任何问题，直接生成了结构化提示词"
- "四段结构：要做什么、怎么做、不能怎么、怎么算做好了"
- "这个模式适合目标明确的用户"

#### 场景二：快速提问模式（2 分钟）

**目的**：展示核心差异化——反向提问挤出隐性意图

```
1. 点击"重新提问"或"← 返回"
2. 输入："这个页面太挤了，帮我优化"
3. 选择"快速提问"模式
4. 点击"开始提问 →"
5. → 进入提问态，展示 7 道核心题
6. 依次回答：
   - "太挤具体指什么？" → 选择"卡片间距"
   - "当前间距大约多少？" → 选择"8px 左右"
   - "目标间距多少？" → 选择"16px"
   - ...（7 题约 1-2 分钟）
7. 回答完毕 → 自动生成提示词
8. → 结果页展示：间距规格已量化为 16px，包含 WCAG 对比度建议
9. 展示底部意图标签清单（实时提取的标签）
```

**讲解要点**：
- "注意引擎实时提取了意图标签——间距、对比度、响应式"
- "评价词典把'太挤'翻译成了'间距从 8px 增加到 16px'"
- "这就是核心差异：不润色你的话，而是问你话"

#### 场景三：截图诊断（1 分钟）

**目的**：展示视觉诊断能力

```
1. 点击"← 返回"回到展开态
2. 拖入一张 UI 截图到截图区域
   （或点击截图区域手动选择图片）
3. → 显示"正在分析截图…"
4. → 1-2 秒后展示诊断报告
5. 展示：检测到的 UI 问题列表（对比度/对齐/间距）
6. 点击"加入提示词"按钮
7. → 诊断标签自动注入意图标签清单
8. → 提示词自动更新，包含截图诊断产出的规格
```

**讲解要点**：
- "Canvas 像素分析，不需要上传到服务器，隐私安全"
- "检测到的问题自动转化为可执行规格"
- "截图标签用黄色区分，来源标注为 screenshot-diagnosis"

#### 场景四：项目感知（30 秒）

**目的**：展示项目上下文自动注入

```
1. 右键悬浮球 → 设置
2. 展示设置页面（LLM 配置、热键自定义、功能开关）
3. 关闭设置
4. 在展开态展示项目绑定区域
5. → 已自动识别当前项目的技术栈（React + Tailwind + Vite）
6. → 提示词中自动注入项目约定（如"使用函数组件"、"Tailwind 类名"）
```

**讲解要点**：
- "扫描项目文件结构和 package.json，自动识别技术栈"
- "生成的提示词会包含项目特定约束，更贴合实际"

#### 场景五：收尾（30 秒）

```
1. 展示提示词历史（最近 3 条）
2. 点击一条历史 → 快速回顾
3. 点击"导出 .md" → 保存为文件
4. ESC → 收起为悬浮球
5. Alt+Shift+Space → 隐藏/显示
```

**讲解要点**：
- "历史记录持久化到 SQLite，重启不丢失"
- "全局热键随时唤起，不打断工作流"
- "安装包仅 3.3MB，轻量常驻"

### 3.3 演示话术总结

> "灵感大王不是一个提示词润色工具。它是一个**意图翻译器**——通过反向提问把你说不清的'感觉'，翻译成 agent 能精准执行的'指令'。
>
> 三档模式覆盖从'只要强化'到'深度诊断'的全场景。评价词典把'太挤'翻译成'间距 16px'。截图诊断把视觉问题转化为规格标签。项目感知让提示词自带技术栈约束。
>
> 所有数据本地存储，API Key 走 Windows 凭据管理器，截图不落盘。3.3MB 安装包，Alt+Shift+Space 随时唤起。"

## 四、使用说明

### 4.1 安装

#### 方式一：安装包（推荐）

1. 获取 `linggandawang_x.x.x_x64-setup.exe`
2. 双击安装（Windows 10 1803+ / Win11）
3. 安装后自动创建桌面快捷方式

#### 方式二：开发模式

```bash
git clone <repo-url>
cd linggandawang
npm install
npm run tauri dev   # 需在独立 PowerShell 窗口执行
```

### 4.2 首次使用

1. **唤起**：`Alt+Shift+Space` 或点击任务栏图标
2. **配置 API Key**（可选）：
   - 右键悬浮球 → 设置
   - 选择服务商（推荐 DeepSeek，性价比高）
   - 填入 API Key → 保存密钥
   - 点击"测试连接"确认可用
3. **开始使用**：单击悬浮球展开输入卡

### 4.3 三档模式选择指南

| 你的情况 | 推荐模式 | 预期耗时 |
|---|---|---|
| 我知道要做什么，只需强化提示词 | 直接生成 | <5 秒 |
| 我有大方向，但关键规格不确定 | 快速提问 | 1-2 分钟 |
| 我只有模糊感觉，需要深度诊断 | 详细诊断 | 5-8 分钟 |

### 4.4 提示词结构说明

生成的提示词包含四段：

| 段 | 标题 | 内容 |
|---|---|---|
| 1 | 要做什么 | 一句话描述目标和场景 |
| 2 | 怎么做 | 3-5 条具体规格，含量化指标 |
| 3 | 不能怎么 | 2-3 条硬约束（项目约定、技术限制） |
| 4 | 怎么算做好了 | 2-3 条可验证的验收标准 |

### 4.5 截图诊断使用

1. 在展开态找到截图区域
2. 拖入图片文件，或点击选择文件
3. 等待 1-2 秒分析完成
4. 查看检测到的问题列表
5. 点击"加入提示词"将问题转化为规格标签

**支持格式**：PNG / JPG / GIF / WebP / BMP / SVG（最大 10MB）

**检测项目**：
- 对比度（WCAG 相对亮度公式）
- 对齐（列投影方差）
- 间距（空白行变异系数）
- 字号（亮度跳变密度分簇）

### 4.6 快捷键

| 快捷键 | 功能 |
|---|---|
| `Alt+Shift+Space` | 全局唤起/隐藏 |
| `ESC` | 收起为悬浮球 |
| `Enter` | 提交输入/答案 |
| 右键悬浮球 | 打开设置 |
| 拖动悬浮球 | 移动位置（自动记忆） |

### 4.7 数据存储位置

| 数据 | 位置 |
|---|---|
| 上下文 | `~/.linggandawang/context.json` |
| 用户偏好 | `~/.linggandawang/user-preferences.json` |
| 窗口位置 | `~/.linggandawang/window-position.json` |
| 自定义题库 | `~/.linggandawang/user-bank.yaml` |
| 提示词历史 | SQLite `linggandawang.db` |
| API Key | Windows Credential Manager（加密存储） |

### 4.8 常见问题

**Q：不配置 API Key 能用吗？**
A：可以。提示词生成完全走规则引擎，不依赖 LLM。API Key 仅用于"测试连接"和未来的 LLM 增强功能。

**Q：截图会上传到服务器吗？**
A：不会。截图分析完全在本地 Canvas 完成，数据不离开内存。

**Q：支持 macOS / Linux 吗？**
A：当前仅支持 Windows。Tauri 框架本身支持跨平台，后续可扩展。

**Q：生成的提示词怎么用？**
A：点击"复制"按钮粘贴到任意 AI 编程工具（Trae / Claude / Cursor / Copilot），或点击"导出 .md"保存为文件。

## 五、技术架构概览

### 5.1 五层分层

```
UI 层          components/          19 个 React 组件
应用逻辑层      hooks/ + stores/     5 hooks + 8 Zustand stores
提问引擎层      engine/              纯 TS 函数库（零依赖）
上下文层        stores/ + lib/       Tauri invoke 读写
平台层          src-tauri/           16 个 Rust 命令
```

### 5.2 数据流

```
用户输入 → seedRouter 路由 → QuestionEngine.start(mode)
  → Selector 选题 → 用户答题 → IntentTagCollector 收集标签
  → PromptGenerator.generate() → 四段结构化提示词
  → 复制/导出/SQLite 持久化
```

### 5.3 状态机

```
ball（悬浮球）⇄ expanded（展开输入卡）
                    ↓
               question（提问态）
                    ↓
                result（结果态）

任意态 + settingsOpen → SettingsPanel（叠加层）
ESC 任意态 → ball
```
