#### 里程碑 M8.6：问题库选项用户化改写（06/24）✅ 已完成

**背景**：用户反馈输入"帮我修改前端风格"时问题跑偏；同时小白用户面对技术选项（Tailwind class / 内边距 16px / WCAG AA / H1-H4）只能乱选。通过行业调研（Typeform 对话式表单原则 + CAT 自适应测试 + RICH 设计范式），确定"用户视角语言 + 参照物"是最佳实践。

**改写原则**：
- `label` 全部改为用户能看懂的口语化描述，加参照物（"像 Linear/Notion/Apple 那样"）
- `tags` 保留技术细节给代码用（如像素值、WCAG 等级、Tailwind），不影响 PromptGenerator 逻辑
- 题干本身也白话化（如"圆角偏好哪种？"→"边角喜欢圆润还是方正？"）
- `placeholder` 改为"或者用你自己的话说..."替代生硬的"或输入..."

**归一化评分公式**：

```
score = 0.4 × tagMatch + 0.3 × cluster + 0.2 × llmConfidence + 0.1 × orderBonus
```

| 维度 | 权重 | 计算方式 | 归一化 |
|------|------|----------|--------|
| tagMatch | 0.4 | 0.6×trigger_tags命中率 + 0.4×tagScores加权 | [0,1] |
| cluster | 0.3 | trigger_clusters命中数 / 总数 | [0,1] |
| llmConfidence | 0.2 | LLM detected_dimensions 加权平均 confidence | [0,1] |
| orderBonus | 0.1 | 1 - (order-1)/(maxOrder-1) | [0,1] |

**边界情况处理**：
- LLM 未配置：Phase 1 降级关键词路由、Phase 3 不触发追问、Phase 4 llmConfidence=0.5 中性

## 九、智能化升级方案（2026-07-15）★ 新增

### 9.1 核心结论

**"灵感大王更智能"≠"调用更大模型"。** 真正的智能化是**分层编排**：

```
规则引擎 + 行为学习（零成本）
  → 本地小模型（隐私 + 离线）
    → RAG（上下文增强）
      → 多 Agent（高级编排）
```

### 9.2 六大方案对比

| 方案 | 工作量 | 成本 | 隐私 | 离线 | 中文 | 可解释 | 推荐 |
|------|--------|------|------|------|------|--------|------|
| A. 本地小模型（SLM） | 3-5d | 零 | ★★★ | ✅ | 中 | 低 | ⭐⭐⭐⭐ |
| B. 规则+模板 | 0.5-1d | 零 | ★★★ | ✅ | — | ★★★ | ⭐⭐⭐⭐⭐ |
| C. 知识图谱 | 100+d | 高 | ★★★ | ✅ | — | ★★★ | ⭐ |
| D. 行为学习 | 1-2d | 零 | ★★★ | ✅ | — | ★★ | ⭐⭐⭐⭐ |
| E. 多 Agent | 5-10d | 中 | ★★ | 部分 | 高 | 中 | ⭐⭐⭐ |
| F. RAG+向量 | 3-5d | 零 | ★★★ | ✅ | 取决于 LLM | 中 | ⭐⭐⭐⭐ |

### 9.3 推荐实施路径

| 阶段 | 时间窗 | 方案 | 投资 | 预期效果 |
|------|--------|------|------|----------|
| **Phase A** | 立即（1-2 周） | B + D | 0 元 / 1.5-3d | "无 LLM"模式质量 +30-50% |
| **Phase B** | 短期（1-2 月） | A（本地 SLM） | 0 元 / 3-5d + 1-2GB 模型 | 80% 场景零成本、离线可用 |
| **Phase C** | 中期（2-3 月） | F（RAG） | 0 元 / 3-5d + 100-200MB 嵌入 | 提示词质量 +20% |
| **Phase D** | 长期（3+ 月） | E（多 Agent） | 中等 / 5-10d | 成本与质量最优平衡 |

### 9.4 决策矩阵

| 场景 | 首选方案 |
|------|----------|
| 用户未配置 API Key | 规则 → 本地 SLM |
| 隐私敏感（项目代码/商业机密） | 本地 SLM + 规则 |

| 跨会话上下文复用 | RAG + 历史答案库 |
| 启动冷启动 | 规则引擎（零配置） |

### 9.6 推荐起点（Phase A 立即可做）

1. **零成本增强**：扩展 seedRouter 添加 5-10 套场景模板（前端 UI / 后端 API / 移动端）
2. **行为学习**：buildProfile() 实现频率统计，跨会话预填
3. **预期**：1.5-3d 工作量，"无 LLM"模式质量提升 30-50%

**为什么不先做本地 SLM？** 因为 Phase A 是数据驱动问题（模板库内容），不是工程问题；先验证规则+行为的极限，再决定是否值得做 SLM 集成（硬件门槛 + 模型分发是真实用户阻力）。
