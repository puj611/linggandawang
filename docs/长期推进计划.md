## 〇、北极星与长期愿景

### 北极星指标（不变）
**「一次模糊灵感 → 挤出的结构化、可执行标签数」** —— 当前均值 7 个，结构化率 100%。

### 长期产品愿景（3 句话）
> 灵感大王不是「提示词润色器」，而是 **vibe coding 新人的「需求显影液」**：
> 用「AI 问、用户答」的反向审问，把脑中混沌的灵感逼成清晰的工程指令；
> 最终成为 **零配置可用、隐私安全、离线可跑、越用越懂你** 的桌面 AI 协作伴侣。

### 长期战略主轴（4 条）
1. **差异化护城河 = 反向审问 + 本地优先**（隐私/离线/零成本，竞品没有）
2. **智能化 ≠ 调更大模型**，而是分层编排（规则→SLM→RAG→多 Agent）
3. **工程质量是复赛/决赛评审的硬通货**（已具 A 级安全基线，补齐测试与 lint 即可开源）
4. **赛后不死**：大赛是放大器，产品本身要能独立存活（分发、社区、口碑）

## 五、阶段Ⅳ：智能化演进期（07/16 – 2027 Q1）

**核心策略**：分层编排，零成本先行，硬件门槛后置。

| 阶段 | 时间窗 | 方案 | 投资 | 预期效果 | 关键集成点 |
|------|--------|------|------|----------|-----------|
| **Phase A** | 07/16-07/31 | B 规则+模板 + D 行为学习 | 0 元 / 1.5-3d | 无 LLM 模式质量 +30-50% | seedRouter 模板库、contextStore.buildProfile() |
| **Phase B** | 09-10/2026 | A 本地 SLM（Qwen2.5 1.5B / Phi-3 Mini） | 0 元 / 3-5d + 1-2GB 模型 | 80% 场景零成本/离线/首字 <100ms | tauri sidecar llama-server、OpenAIAdapter 双 baseUrl |
| **Phase C** | 11-12/2026 | F RAG + 本地向量（BGE-small-zh） | 0 元 / 3-5d + 100-200MB | 提示词质量再 +20%、跨会话复用 | vectorStore.ts、TemplateRetriever.ts |
| **Phase D** | 2027 Q1 | E 多 Agent 编排 | 中等 / 5-10d | 成本质量最优平衡、Pro 功能 | 调度层：本地 SLM 分类 → 云端 LLM / 规则兜底 |

**决策矩阵（何时走哪条路）**：

| 场景 | 首选 |
|------|------|
| 用户未配 API Key | 规则 → 本地 SLM |
| 隐私敏感（项目代码/商业机密） | 本地 SLM + 规则 |
| 离线/出差 | 本地 SLM |
| 复杂需求深度拆解 | 云端 LLM（已有路径） |
| 跨会话上下文复用 | RAG + 历史答案库 |
| 冷启动 | 规则引擎（零配置） |

**为什么 Phase B 不在复赛前做？** 硬件分发（1-2GB 模型）+ 首次下载体验是真实用户阻力，且 Phase A 是「数据驱动问题」，先验证规则+行为极限，再决定是否值得做 SLM 集成。

## 十一、一句话总结

**灵感大王长期计划 = 「复赛稳、决赛冲、赛后活」三段式**：
复赛期用零成本智能化（Phase A）提分 + 还工程债（P0）；
