## 0. 一句话概述

> **灵感大王是一个桌面悬浮窗工具，它不润色你的提示词，而是通过反向提问把模糊灵感「挤」成 agent 能精准执行的结构化指令——专治 vibe coding 新人「有想法说不清、AI 润色反而失真、agent 改不对或没改」的痛点。**

---

## 1. 项目概述

### 1.1 痛点根源

vibe coding 新人的核心痛点不是「不会写代码」，而是**意图传达损耗**——隐性意图在向显性指令传递时持续衰减。具体表现为四类断裂：

| 痛点 | 本质 | 传统做法的失败 |
|------|------|---------------|
| 找不到灵感 | 素材库未建立，无组合起点 | 让 AI 直接出方案 → 得到平均值 |
| AI 润色后失真 | AI 用统计默认值补全你没说的部分 | 越润色越偏离原意 |
| 感觉不够好却说不清 | 审美判断先于语言，缺词汇脚手架 | 只能说「差点意思」→ 无法执行 |
| agent 改不对/没改 | 指令落在「评价/方向」而非「动作/规格」 | agent 自行决策 → 偏离或不动 |

**统一根因**：传统做法试图「描述得更好」，但隐性知识天然难言传；真正的解法是**换一套传递机制**——用参照物代替形容词、用提问代替润色、用结构化模板把「翻译」拆成可校验的小步。

### 1.2 产品定位

- **形态**：桌面悬浮窗（常驻、轻量、一键唤起）
- **目标用户**：vibe coding 新人，特别是有想法但说不清的开发者
- **核心价值**：把「感觉不对」翻译成「精准指令」
- **差异化**：**不润色，而是提问引导 + 视觉诊断 + 结构化模板**
- **第一阶段**：端侧 Demo（MVP），完全离线可用，提问引擎用本地规则模板
- **后续阶段**：云端部署、接入 LLM、跨设备同步

### 1.3 核心机制（与传统润色的本质区别）

| 维度 | 传统润色工具 | 灵感大王 |
|------|------------|---------|
| 交互范式 | 你写一段，AI 改一段 | AI 先问你，你再答 |
| 信息流向 | AI 补全你没说的（用统计默认值） | AI 暴露你没说的（让你说） |
| 产物 | AI 的平均值 | 你的特异性，被结构化 |
| 动作粒度 | 一次性大动作 | 多轮小步对齐 |
| 失真风险 | 高（润色越多越偏） | 低（你说的才是最终产物） |

### 1.4 统一框架：意图传达管线

灵感大王的产品逻辑建立在一个统一框架上——**意图传达管线**，五阶段每阶段配对策：

```
感知(Perceive) → 命名(Name) → 规格(Specify) → 执行(Execute) → 验证(Verify)
   │                │              │                │              │
 参照物锚定      维度脚手架      目标定位        结构化变更      截图闭环
 +AI 反向审问    +视觉诊断       +验收标准       +反向复述       +diff 迭代
```

## 3. 架构设计摘要

> 完整架构设计见 `docs/架构设计.md`（高见远产出，11 章节，含 Mermaid 图与 30 条任务列表）。此处摘要核心。

### 3.1 技术选型

| 技术 | 版本 | 用途 | 选型理由 |
|------|------|------|---------|
| Tauri | 2.0 | 桌面壳 | 包体≤10MB，参赛轻量加分 |
| React | 18 | UI | 生态成熟，与现有技术栈契合 |
| TypeScript | 5.x | 类型安全 | 工程化基础 |
| Tailwind CSS | 3.x | 样式 | 原子化，视觉 token 易维护 |
| Zustand | 4.x | 状态管理 | 轻量，适合中小型应用 |
| Vite | 5.x | 构建 | HMR 快，Tauri 官方推荐 |
| Rust | - | 平台层 | 窗口/热键/文件/图像（Demo 阶段降级） |

### 3.2 分层架构

```
UI 层（React + Tailwind）         4 态 13 组件，只画不存
   │
应用逻辑层（Hooks + Zustand）     5 store + useFlow 编排
   │
提问引擎层（纯 TS 函数库）        零副作用，可单测可热更新 ← 产品差异化核心
   │
上下文层（TS + Rust 混合）        context.json 读写 + 24h 归档
   │
平台层（Rust + Tauri）            窗口/热键/文件/图像（Demo 降级到前端）
```

**原则**：前端能做的不上 Rust。提问引擎是纯函数库，零外部依赖，便于单测与热更新。

### 3.3 六大核心模块

| 模块 | 职责 | 关键设计 |
|------|------|---------|
| M1 WindowManager | 置顶/拖动/四态切换/热键 | mouse 事件降级（Rust 不可用） |
| M2 QuestionEngine | 状态机 + 动态选题 + 跳过 | 5 阶段 FSM + 三级选题（jump>trigger_tags>兜底） |
| M3 PromptGenerator | 四段拼装 + 评价展开 + 导出 | 评价词典 8 关键词 → 量化规格 |
| M4 ContextStore | context.json 读写 + 归档 | localStorage 降级 + 24h 过期 |
| M5 ScreenshotDiagnoser | 对比度/对齐/间距/字号 | Canvas WCAG + 双开关降级 |
| M6 IntentTagCollector | 标签累计 + chip + 删除重生成 | 北极星埋点 |

### 3.4 提问引擎设计（产品差异化核心）

- **状态机**：IDLE → PERCEIVE → NAME → SPEC → EXECUTE → VERIFY → COMPLETE
- **选题规则**：阶段过滤 → 去重 → trigger_tags 命中 → jump 强制 → 兜底 order
- **种子路由**：12 条关键词规则，把模糊种子路由到首问题（如"太挤" → 间距类问题）
- **问题库**：YAML 文件，支持热更新（内置 bank.yaml + 用户目录覆盖）
- **评价词典**：8 个痛点关键词 → 量化规格映射（"太挤"→"内边距16px"、"不够高级"→"字号层级+留白"等）

### 3.5 截图诊断降级机制（D2 go/no-go 闸门）

- **双开关**：编译期 Cargo feature flag + 运行时 config.json 开关
- **W3 末评审**：准确率 < 60% 一键降级
- **降级后**：截图按钮置灰 + 提问引擎插入兜底问题"请用 3 个词描述你看到的问题"
- **主流程不中断**：截图诊断是加分项，不是闭环必经节点
- **QA 建议**：**GO**（对比度 WCAG 完整，其他简化但有兜底，双开关机制完善）

### 4.3 降级方案说明

环境限制（Rust/Cargo 不可用）导致 19 处降级，全部用 `// DOWNGRADE` 注释标注：

| 模块 | 原计划（Rust/Tauri） | 降级实现 | 影响 |
|------|---------------------|---------|------|
| 桌面壳 | Tauri 窗口 | Vite 浏览器全屏 div | 失去桌面形态，但核心闭环可用 |
| 全局热键 | tauri-plugin-global-shortcut | document keydown | 仅前台生效 |
| 拖动持久化 | Rust fs | localStorage | 跨浏览器不保留 |
| context.json | Rust std::fs | localStorage | 跨工具同步降级为同浏览器 |
| 截图诊断 | Rust image crate | Canvas getImageData | 对比度完整，其他简化 |
| 文件导出 | Rust fs | Blob + `<a download>` | 功能等价 |

**降级不阻断核心闭环**：唤起 → 提问 → 生成 → 复制 全流程可用。Phase 2 补 Rust 工具链后可恢复完整桌面形态。

## 7. 参赛亮点

### 7.1 核心叙事：从「润色」到「提问」的范式反转

灵感大王的市场定位独占「主动引导 + 垂直场景」象限，**无直接竞品**。所有现有工具（PromptPerfect、AIPRM、Cursor、v0.dev、ChatGPT 等）都是「被动润色」——用户写、AI 改。灵感大王是唯一「主动引导」——AI 问、用户答。这是大赛最强的差异化叙事。

### 7.2 五大可演示亮点

1. **反向提问引擎**：5 阶段状态机 + 动态选题 + 种子路由，把"卡片太挤"自动路由到间距类问题
2. **评价转规格**：8 个痛点关键词自动展开为量化规格（"太挤"→"内边距16px"），对治 agent 改不对
3. **四段结构化提示词**：动作/规格/约束/验证，100% 结构化率，agent 不会漏改
4. **截图视觉诊断**：Canvas WCAG 对比度 + 兜底问题，对治"感觉不对说不清"
5. **意图标签累计**：单次提问挤出 7 个标签（北极星目标 3 个，远超 133%）

### 7.3 数据支撑

| 指标 | 目标 | 实测 | 亮点 |
|------|------|------|------|
| 标签挤出数 | ≥3 | 7 | 远超 133% |
| 全流程耗时 | <60s | ≈30s | 快 50% |
