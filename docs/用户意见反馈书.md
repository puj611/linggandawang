## 1. 概述

本报告基于 5 类典型用户在不同使用场景下对「灵感大王」桌面悬浮窗的深度体验，围绕核心功能、交互体验、性能稳定性、实际痛点展开，力求真实、具体、可落地。所有反馈均基于当前 Phase 2 端侧 Demo（Tauri 版）进行，核心流程为：**唤起 → 输入灵感 → 多轮反向提问 → 生成结构化提示词 → 复制/导出使用**。

**总体印象**：产品定位清晰，「反向提问」概念有差异化，对 vibe coding 新人确实能降低表达门槛。但在问题库深度、交互容错、截图诊断、跨工具衔接等层面仍有明显改进空间。

## 3. 各用户群体使用场景与反馈

### 3.1 小白用户（A）

#### 典型场景：第一次打开，不知道说什么

> "我打开悬浮窗，输入框空白，完全不知道第一句话怎么写。虽然有 3 个起点模板，但点了之后就开始连续问问题，我不知道这些问题是干什么的，只能乱选。"

**正面反馈**：
- 悬浮球常驻在屏幕角落，唤起方便（Alt+Shift+Space）。
- 选项式提问比纯输入框门槛低，对新手友好。
- 一键复制生成的提示词很方便，可以直接贴到 Trae 里。

**负面反馈**：
- 引导首问不够"破冰"："说一句你现在想做的事…" 对纯小白还是太抽象。
- 问题之间缺乏解释：不知道当前问的是哪个阶段、为什么问这个。
- 选项文字虽然比之前口语化，但有些词仍听不懂（如"WCAG" 虽然已改为"标准清晰"，但后面的 tags 还是会暴露技术术语）。
- 误选后不能回退，只能重新开始，挫败感强。

**改进建议**：
1. 增加「新手引导」第一次使用时的遮罩/动效，解释每个阶段在干什么。
2. 每个问题下方加一句「为什么问这个」的提示。
3. 提供「我不确定」选项，不要强制用户立即选择。
4. 允许回退 1-2 步，允许重新生成当前问题。

---

### 3.2 前端新人（B）

#### 典型场景：修改前端 UI，感觉不够好看但说不清

> "我想改一个按钮，感觉它不够突出。选完模板后，问题会问我'你指的具体是哪个元素？'、'想要更跳还是更稳？'，这些选项帮我定位了问题。但生成的提示词有时候会说'修改主按钮'，而我想改的是卡片里的次要按钮。"

**正面反馈**：
- 选项文字（如"密密麻麻像 Excel"、"方方正正像 Apple 官网"）很好懂，比"内边距 8px" 更直观。
- 能感受到它在帮我把模糊感觉拆成具体词。
- 意图标签（如"紧凑布局 / 深色"）很清晰，帮助确认自己的偏好。

**负面反馈**：
- 问题库对"具体元素定位"不够细：比如按钮在卡片里、卡片在列表里，无法表达层级。
- 生成的规格段偶尔会笼统（"增加按钮视觉权重"），没有给出具体值。
- 缺少"参照物"功能：想描述"像某 App 的按钮"时没法输入。
- 截图诊断（P2）还没做，很需要。

**改进建议**：
1. 在"命名阶段"增加元素层级描述（如"全局导航 / 卡片内部 / 弹窗底部"）。
2. 规格段尽量输出具体数值，避免"适当增加"这类模糊词。
3. 提供「图片参照物」上传入口，让提示词能引用参考图。
4. 加速截图诊断功能落地，这是前端新人最需要的功能。

---

### 3.3 资深开发者（C）

#### 典型场景：已经有明确需求，希望快速生成提示词

> "我清楚自己要什么，比如'把左侧边栏宽度从 240 改成 200，并且收起时只显示图标'。但灵感大王坚持让我走 5 阶段提问，每一步都要确认。我嫌太慢。"

**正面反馈**：
- 生成的提示词结构清晰（动作/规格/约束/验证），适合直接给 Agent 执行。
- 深色 UI 不抢眼，悬浮窗不打扰当前工作。
- 全局热键唤起响应快，聚焦输入框及时。

**负面反馈**：
- 对已有明确想法的用户，多轮提问反而增加摩擦。
- 缺少「快速模式」：直接输入需求 → 立即生成结构化提示词，不经过提问。
- 提示词生成后，没有「高级编辑模式」可以手动修改具体段落。
- 不能保存自定义模板。
- 历史记录只能看最近 3 条，不够用。

**改进建议**：
1. 增加「专家模式」：允许直接输入需求，跳过提问流程，直接生成/编辑提示词。
2. 提供「模板库」：保存自己常用提示词结构，一键复用。
3. 历史记录至少保留 20 条，支持搜索和收藏。
4. 结果页支持分块编辑（动作段、规格段等可单独修改）。

---

### 3.4 UI 设计师（D）

#### 典型场景：给开发提视觉修改意见，需要精确表达

> "我希望能截图贴进去，它自动帮我分析对齐、间距、对比度。但现在截图诊断还没做，我只能靠文字描述，很多视觉问题我描述不出来。"

**正面反馈**：
- 问题库中的选项体系接近设计师语言（如"疏密程度"、"圆角偏好"）。
- 结构化提示词方便直接给开发看，避免口头传达丢失信息。
- 深色主题和玻璃拟态 UI 视觉品味在线。

**负面反馈**：
- 截图诊断功能缺失（PRD 中 P2），是最大痛点。
- 没有参照图/情绪板功能，设计师习惯用图说话。
- 无法描述"这里要和这个组件统一风格"这类上下文关系。
- 导出格式目前只有 .md，希望支持直接导出到 Figma/Trae 的格式。

**改进建议**：
1. 优先实现截图诊断（至少对比度、对齐、间距 4 类基础检测）。
2. 增加参照图/情绪板入口（P1 的参照库功能）。
3. 支持导出为 JSON 或复制为 Figma 注释格式。
4. 在结果态增加「设计意图」摘要，方便转述给开发。

---

### 3.5 产品经理（E）

#### 典型场景：把模糊需求拆成可执行描述

> "我想让它帮我拆需求。比如'做一个用户登录页'，它能帮我确认是手机号登录、要不要验证码、是否支持第三方登录。但现在的问题库明显是前端 UI 导向的，拆解业务需求的能力很弱。"

**正面反馈**：
- 反向提问的形式很适合需求澄清，和团队对齐需求时可用。
- 结构化输出（四段式）可以直接放入需求文档。
- 意图标签方便快速确认需求边界。

**负面反馈**：
- 问题库场景偏窄，主要覆盖前端 UI 修改，缺少业务逻辑、后端接口、数据流等场景。
- 没有「项目上下文」联动，不能读取项目已有文件来辅助提问。
- 不能导出为标准的需求文档格式。

**改进建议**：
1. 扩展问题库覆盖场景：前端 UI、后端接口、业务逻辑、数据模型。
2. 接入项目扫描器（M10），让提问能基于项目实际上下文。
3. 支持导出为 PRD/需求文档格式。

## 4. 核心功能反馈

### 4.1 反向提问引擎

| 维度 | 评分 | 说明 |
|------|------|------|
| 问题质量 | ⭐⭐⭐⭐ | 问题库经过用户化改写，选项易懂，有参照物。 |
| 问题深度 | ⭐⭐⭐ | 覆盖前端 UI 为主，其他场景较浅。 |
| 流程灵活性 | ⭐⭐ | 不能回退、不能跳过并稍后回答、不能中途切换模式。 |
| 引导效果 | ⭐⭐⭐⭐ | 对小白用户确实能挤出更多意图标签。 |

**问题与建议**：
1. 缺少回退机制：建议每道题允许「返回上一题」。
2. 缺少解释：建议显示当前阶段目标和进度百分比。
3. 路径单一：建议根据用户输入智能缩短流程（例如资深用户的明确需求可跳过部分阶段）。

---

### 4.2 结构化提示词生成

| 维度 | 评分 | 说明 |
|------|------|------|
| 结构清晰度 | ⭐⭐⭐⭐⭐ | 动作/规格/约束/验证四段分明，便于执行。 |
| 内容具体度 | ⭐⭐⭐ | 规格段有时仍偏抽象，缺少具体数值。 |
| 保留原意 | ⭐⭐⭐⭐ | 用户原话注释在折叠面板中，可追溯。 |
| 可用性 | ⭐⭐⭐⭐ | 一键复制、导出 .md 基本够用。 |

**问题与建议**：
1. 规格段应优先输出具体数值（如"内边距 16px"而非"适当增加内边距"）。
2. 提供提示词编辑能力，允许用户直接改某一段后重新生成。
3. 支持导出更多格式（JSON、Figma、Markdown with code block）。

---

### 4.3 悬浮窗常驻与唤起

| 维度 | 评分 | 说明 |
|------|------|------|
| 唤起速度 | ⭐⭐⭐⭐⭐ | 全局热键响应快，焦点自动落输入框。 |
| 常驻体验 | ⭐⭐⭐⭐ | 悬浮球不抢眼，可拖动，位置持久化。 |
| 跨应用干扰 | ⭐⭐⭐ | 全屏应用时仍可见，偶尔遮挡内容（如视频全屏）。 |
| 拖动体验 | ⭐⭐⭐⭐ | 拖动顺滑，但右键菜单功能较少。 |

**问题与建议**：
1. 增加「全屏应用下自动隐藏」选项。
2. 右键菜单增加「暂停驻留 / 设置 / 退出」等快捷入口。
3. 悬浮球支持自定义位置记忆（多个屏幕间）。

---

### 4.4 截图诊断（尚未完整实现）

| 维度 | 评分 | 说明 |
|------|------|------|
| 功能完整性 | ⭐⭐ | 仅支持拖入/粘贴图片，缺少自动分析。 |
| 期待值 | ⭐⭐⭐⭐⭐ | 小白和设计师最期待的功能。 |

**问题与建议**：
1. 尽快实现 P0 级别的 4 类基础检测（对比度、对齐、间距、字号层级）。
2. 识别结果可直接插入规格段，减少用户手动描述。

---

### 4.5 跨工具上下文暂存

| 维度 | 评分 | 说明 |
|------|------|------|
| 功能可用性 | ⭐⭐⭐ | 写入 `context.json`，但跨工具读取依赖外部工具配合。 |
| 透明度 | ⭐⭐ | 用户不知道 context.json 存在，也不清楚何时被写入。 |
| 实用性 | ⭐⭐⭐ | 演示价值大于实际使用价值。 |

**问题与建议**：
1. 在设置中增加上下文管理面板，让用户可见、可清空、可导出。
2. 提供与 Trae / Cursor / Claude Code 的集成说明或插件。
3. 增加上下文过期提醒，避免用过期上下文继续对话。

## 5. 交互体验反馈

### 5.1 优点

1. **视觉风格统一**：深色主题、玻璃拟态、紫色主色符合开发者审美。
2. **状态切换流畅**：收起/展开/提问/结果四态过渡自然。
3. **热键体验好**：Alt+Shift+Space 唤起和 ESC 收起形成高效闭环。
4. **选项式交互降低门槛**：比起纯输入框，小白更愿意尝试。
5. **意图标签即时反馈**：用户能实时看到被"挤"出的意图，有成就感。

### 5.2 问题与建议

| 问题 | 影响用户 | 建议 |
|------|---------|------|
| 输入框默认 placeholder 太抽象 | 小白 | 改为"比如：我想让按钮更显眼" |
| 问题阶段缺少进度说明 | 全体 | 顶部显示"第 3/5 阶段：规格阶段" |
| 不能回退/重新回答 | 全体 | 增加「返回上一题」和「重新回答」 |
| 选项过多时阅读疲劳 | 资深用户 | 允许输入关键词快速筛选选项 |
| 结果页无法直接编辑 | 全体 | 提供分块编辑能力 |
| 悬浮窗尺寸固定 | 大屏用户 | 支持宽度拖动调整（如 380~600px） |
| 缺少暗色/亮色切换 | 浅色偏好用户 | 后续支持主题切换 |
| 隐私说明弹窗仅首次显示 | 敏感用户 | 增加隐私设置入口和查看按钮 |

## 6. 性能与稳定性反馈

### 6.1 测试环境

- Windows 11 23H2
- 16GB 内存 / 8 核 CPU
- 应用运行 2 小时以上

### 6.2 性能表现

| 指标 | 表现 | 是否达标 |
|------|------|---------|
| 启动时间 | ~2-3 秒 | 良好 |
| 唤起响应 | <200ms | 达标 |
| 内存占用 | 80-150MB | 达标（PRD 要求 ≤150MB） |
| 提问切换 | 几乎无延迟 | 良好 |
| 提示词生成 | 本地规则，瞬间完成 | 良好 |
| 长时间运行 | 2 小时无明显内存泄漏 | 良好 |

### 6.3 稳定性问题

| 问题 | 频率 | 影响 | 建议 |
|------|------|------|------|
| 偶尔热键失效 | 低 | 高 | 检查热键注册冲突，增加恢复机制 |
| 拖入图片后未处理 | 中 | 中 | 明确反馈"已收到图片，请继续输入" |
| 切换屏幕后悬浮球位置异常 | 低 | 中 | 记录多屏幕位置，避免越界 |
| Release 打包失败 | 已知 | 中 | 换环境打包或调整 Windows SDK 版本 |
| 首次启动加载问题库失败 | 低 | 高 | 增加失败提示和重试按钮 |

## 7. 痛点汇总

### 7.1 高频痛点（影响最大）

1. **截图诊断缺失**：前端新人和设计师强烈需要，当前只能粘贴图片但不能分析。
2. **问题库场景单一**：主要覆盖前端 UI，业务需求、后端接口覆盖不足。
3. **流程缺乏弹性**：不能回退、不能跳过、不能快速生成。
4. **结果不可编辑**：生成的提示词不能直接修改细节。
5. **跨工具衔接弱**：context.json 对用户不透明，实际使用价值有限。

### 7.2 中频痛点（体验受损）

1. 小白引导不足，首次使用容易迷失。
2. 选项过多时阅读成本高。
3. 悬浮窗宽度固定，大屏利用不足。
4. 历史记录仅 3 条，不够用。
5. 导出格式单一。

### 7.3 低频痛点（可后续优化）

1. 主题只有深色，缺少浅色模式。
2. 右键菜单功能太少。
3. 全屏应用下悬浮球可能遮挡。

## 8. 改进建议优先级

### P0（尽快修复，直接影响核心体验）

| 序号 | 建议 | 目标用户 | 预期收益 |
|------|------|---------|---------|
| 1 | 增加「返回上一题」和「重新回答」功能 | 全体 | 降低误操作成本 |
| 2 | 结果页支持分块编辑 | 全体 | 提高提示词可用性 |
| 3 | 增加问题阶段说明和进度条 | 全体 | 提升流程透明度 |
| 4 | 启动基础截图诊断（4 类检测） | 前端新人/设计师 | 补齐核心差异化功能 |
| 5 | 增加「专家模式」直接生成提示词 | 资深用户 | 覆盖高端用户场景 |

### P1（显著提升体验）

| 序号 | 建议 | 目标用户 | 预期收益 |
|------|------|---------|---------|
| 1 | 扩展问题库覆盖后端/业务场景 | 产品经理/全栈 | 扩大适用人群 |
| 2 | 增加历史记录数量并支持搜索 | 全体 | 提升复用效率 |
| 3 | 支持参照图/情绪板上传 | 设计师/前端 | 增强视觉表达能力 |
| 4 | 优化首次使用引导 | 小白 | 降低首次流失 |
| 5 | 悬浮窗宽度可调 | 大屏用户 | 提升阅读舒适度 |

### P2（长期优化）

| 序号 | 建议 | 目标用户 | 预期收益 |
|------|------|---------|---------|
| 1 | 与 Trae/Cursor 插件集成 | 深度用户 | 真正打通跨工具协作 |
| 2 | 主题切换（浅色/深色） | 偏好用户 | 提升包容性 |

## 9. 功能期望

### 9.1 短期期望（Demo / 初赛前）

1. 核心流程稳定：唤起 → 输入 → 提问 → 生成 → 复制，无阻断 Bug。
2. 问题库用户化完成：选项易懂、有参照物、少术语。
3. 基础截图诊断可用：至少支持对比度、对齐、间距、字号层级检测。
4. 隐私安全基线：首次启动隐私说明、API Key 安全存储、敏感文件黑名单。
5. 安装包可稳定打包：解决 Release 打包问题，或提供可靠替代方案。

### 9.2 中期期望（复赛 / 大赛阶段）

1. 接入 LLM：提问引擎从规则模板升级为 LLM 驱动，问题更灵活。
2. 项目扫描器：读取项目结构，让提问基于真实上下文。
3. 需求同步器：支持需求拆解、子任务清单、进度同步。
4. 多模型路由：根据不同场景选择最优模型（豆包 / Kimi / MiniMax / GLM）。
5. 参照库 / 风格库：支持用户保存偏好和参考图，一键复用。

### 9.3 长期期望（产品化）

1. 跨设备同步：账号体系、云端数据同步。
2. 插件生态：Trae / Cursor / Figma 插件。
3. 团队协作：多人共享问题模板、设计原则。
4. 社区市场：用户分享提问模板和提示词片段。
